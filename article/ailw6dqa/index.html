rpc:call和genserver:call有什么区别

约 260 字小于 1 分钟

erlangrpcgen_ser

2025-03-28

提示

rpc:call 和 gen_server:call 都是 Erlang 中用于进行进程间通信的机制

使用场景

  • rpc:call 适用于跨节点的远程过程调用，能够让你在分布式系统中的任意节点上调用其他节点的进程。

  • gen_server:call 适用于同一节点内的进程间同步通信，通常用于本地服务进程的调用。

调用方式

rpc:call

相关信息

需要指定远程节点 (Node)，模块 (Module)，函数 (Function)，以及参数 (Args)。调用会发送到远程节点，并且是同步的。

-spec call(Node, Module, Function, Args) -> Res | {badrpc, Reason} when
      Node :: node(),
      Module :: module(),
      Function :: atom(),
      Args :: [term()],
      Res :: term(),
      Reason :: term().

gen_server:call

相关信息

调用时需要指定目标进程（通常是一个 gen_server 进程的注册名或 PID）和请求数据 (Request)。gen_server:call 会将请求发送到目标进程，并等待该进程的响应。

-doc(#{equiv => call(ServerRef, Request, 5000)}).
-spec call(
        ServerRef :: server_ref(),
        Request   :: term()
       ) ->
                  Reply :: term().
