静态文件

约 1227 字大约 4 分钟

2025-03-28

Cowboy 提供了一个用于提供静态文件的内置处理程序。在开发过程中，这可以作为方便的解决方案。

对于生产系统，考虑使用市场上提供的许多内容分发网络（CDN）之一，因为它们是提供文件的最佳解决方案。

静态处理程序可以提供一个文件或指定目录下的所有文件。可以配置 etag 生成和 mime 类型。

提供一个文件

您可以使用静态处理程序从应用程序的私有目录中提供一个特定的文件。当客户端请求根路径（例如 /）时，这特别适用于提供一个 index.html 文件。配置的路径相对于给定的应用程序的私有目录。

以下规则会在访问路径 / 时，从应用程序 my_app 的 priv 目录提供文件 static/index.html：

{"/", cowboy_static, {priv_file, my_app, "static/index.html"}}

你也可以指定文件的绝对路径，或者相对于当前目录的路径：

{"/", cowboy_static, {file, "/var/www/index.html"}}

从一个目录提供所有文件

你也可以使用静态处理程序来提供在配置目录中找到的所有文件。处理程序将使用 path_info 信息来解析文件位置，这意味着你的路由必须以 [...] 模式结束才能生效。所有文件都会被提供，包括可能在子文件夹中找到的文件。

你可以指定相对于应用程序的私有目录的目录（例如 my_app/priv）。

以下规则将在请求路径以 /assets/ 开始时，提供 my_app 应用程序的私有目录中的 my_app/priv/static/assets 文件夹中的任何文件：

{"/assets/[...]", cowboy_static, {priv_dir, my_app, "static/assets"}}

你也可以指定绝对路径的目录，或者将其设置为相对于当前目录的路径：

{"/assets/[...]", cowboy_static, {dir, "/var/www/assets"}}

自定义 MIME 类型检测

默认情况下，Cowboy 会尝试通过查看文件扩展名来识别静态文件的 MIME 类型。

你可以覆盖用于确定静态文件 mimetype 的函数。当 Cowboy 缺少你需要处理的 mimetype 时，或者你想减少列表以加快查找速度时，这会很有用。你也可以提供一个硬编码的 mimetype，它将无条件使用。

Cowboy 内置了两个函数。默认函数仅处理构建 Web 应用程序时常用的一些文件类型。另一个函数则包含数百种 mimetype，几乎可以满足你的所有需求。当然，你也可以创建自己的函数。

要使用默认函数，你不需要进行任何配置，因为它已经是默认设置。不过，如果你坚持要配置，以下代码可以完成任务：

{"/assets/[...]", cowboy_static, {priv_dir, my_app, "static/assets",
    [{mimetypes, cow_mimetypes, web}]}}

你可以看到，有一个可选字段，可能包含一些较少使用的选项，比如 mimetypes 或 etag。所有选项类型都有这个可选字段。

要使用可以检测几乎任何 MIME 类型的功能，以下配置即可：

{"/assets/[...]", cowboy_static, {priv_dir, my_app, "static/assets",
    [{mimetypes, cow_mimetypes, all}]}}

您可能现在已经注意到了这个模式。配置期望一个模块和一个函数名，因此您可以使用您自己的任何函数代替：

{"/assets/[...]", cowboy_static, {priv_dir, my_app, "static/assets",
    [{mimetypes, Module, Function}]}}

用于执行 mimetype 检测的函数接收单个参数，即文件在磁盘上的路径。建议以元组形式返回 mimetype，虽然也可以返回二进制字符串（但需要额外处理）。如果函数无法确定 mimetype，则应返回 {<<"application">>, <<"octet-stream">>, []} 。

当静态处理程序找不到扩展名时，它将发送文件为 application/octet-stream。浏览器接收到此类文件时，将尝试直接将其下载到磁盘。

最后，可以为所有文件硬编码 mimetype。特别是在与 file 和 priv_file 选项结合使用时，这可以避免不必要的计算：

{"/", cowboy_static, {priv_file, my_app, "static/index.html",
    [{mimetypes, {<<"text">>, <<"html">>, []}}]}}

生成一个 etag

默认情况下，静态处理程序会根据文件大小和修改时间生成一个 etag 头值。不过，这种解决方案并不适用于所有系统。例如，在节点集群中，这种方案的表现会相当差，因为文件元数据在每台服务器上都会有所不同，导致每台服务器上的 etag 值都不同。

然而，你可以改变 etag 的计算方式：

{"/assets/[...]", cowboy_static, {priv_dir, my_app, "static/assets",
    [{etag, Module, Function}]}}

该函数将接收三个参数：文件在磁盘上的路径、文件大小以及最后修改时间。在分布式环境中，你通常会使用文件路径来获取一个在所有服务器上都相同的 etag 值。

你也可以完全禁用 etag 处理：

{"/assets/[...]", cowboy_static, {priv_dir, my_app, "static/assets",
    [{etag, false}]}}
