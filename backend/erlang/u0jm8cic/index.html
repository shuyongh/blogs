监听器

约 617 字大约 2 分钟

2025-03-28

提示

一个监听器是一组进程，它们在端口上监听新的连接请求。新的连接请求将由 Cowboy 处理。根据连接握手，可能会使用不同的协议。

本章仅适用于 Cowboy。请参阅 Ranch 用户指南以获取有关监听器的更多信息。

Cowboy 提供了两种类型的监听器：

  • 一种用于监听清晰的 TCP 连接 (HTTP)
  • 另一种用于监听安全的 TLS 连接。(HTTPS)

这两种监听器都支持 HTTP/1.1 和 HTTP/2 协议。

清晰的 TCP 监听器--HTTP

清晰的 TCP 监听器将在给定端口接受连接。

start(_Type, _Args) ->
    Dispatch = cowboy_router:compile([
        {'_', [{"/", hello_handler, []}]}
    ]),
    {ok, _} = cowboy:start_clear(my_http_listener,
        [{port, 8080}],
        #{env => #{dispatch => Dispatch}}
    ),
    hello_erlang_sup:start_link().

连接到 Cowboy 清晰监听器端口的客户端应使用 HTTP/1.1 或 HTTP/2。

牛仔支持通过升级机制（RFC 7540 3.2）或直接发送前缀（RFC 7540 3.4）来建立清晰的 HTTP/2 连接。

通过牛仔的 HTTP/1.1 实现提供与 HTTP/1.0 的兼容性。

安全的 TLS 监听器--HTTPS

安全的 TLS 监听器将在给定端口接受连接。

Cowboy 提供的功能将确保给定的 TLS 选项遵循 HTTP/2 RFC 中的安全要求。例如，某些 TLS 扩展或加密算法可能会被禁用。这也适用于此监听器上的 HTTP/1.1 连接。如果不需要这样做，可以直接使用 Ranch 来设置自定义监听器。

start(_Type, _Args) ->
    Dispatch = cowboy_router:compile([
        {'_', [{"/", hello_handler, []}]}
    ]),
    {ok, _} = cowboy:start_tls(my_https_listener,
        [
            {port, 8443},
            {certfile, "/path/to/certfile"},
            {keyfile, "/path/to/keyfile"}
        ],
        #{env => #{dispatch => Dispatch}}
    ),
    hello_erlang_sup:start_link().

注

连接到 Cowboy 的客户端应使用ALPN TLS扩展来表明他们支持哪些协议。Cowboy 总是优先使用 HTTP/2 而不是 HTTP/1.1，当两者都支持时。当客户端不支持这两种协议，或者缺少 ALPN 扩展时，Cowboy 期望使用 HTTP/1.1。

停止监听器

在启动监听器的同时启动应用程序时，最好在应用程序停止时也停止监听器。这可以通过在应用程序的停止函数中调用 cowboy:stop_listener/1 来实现。

stop(_State) ->
    ok = cowboy:stop_listener(my_http_listener).

协议配置

HTTP/1.1 和 HTTP/2 协议具有相同的语义；只有它们的帧格式不同。前者是文本协议，后者是二进制协议。

牛仔不会为 HTTP/1.1 和 HTTP/2 分开配置。所有内容都放入同一个映射中。许多选项是共享的。
