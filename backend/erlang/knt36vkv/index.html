发送响应

约 2132 字大约 7 分钟

2025-03-28

响应必须使用 Req 对象发送。

Cowboy 提供了两种发送响应的方法：直接发送或流式传输响应体。响应头和响应体可以在发送前设置。当调用回复或流式回复函数时，响应将立即发送。

Cowboy 还提供了一个简化接口用于发送文件。它还可以发送文件的特定部分。

虽然每个请求只能有一个响应，但 HTTP/2 引入了一种机制，允许服务器推送与响应相关的其他资源。本章还将介绍 Cowboy 中此功能的工作方式。

回复

Cowboy 提供了三个函数来发送整个回复，具体取决于您是否需要设置头部和正文。在所有情况下，Cowboy 都会添加协议所需的任何头部（例如，日期头部总是会被发送）。

如果您只需要设置状态码，请使用 cowboy_req:reply/2：

Req = cowboy_req:reply(200, Req0).

当您需要同时设置响应头时，请使用 cowboy_req:reply/3：

Req = cowboy_req:reply(303, #{
    <<"location">> => <<"https://ninenines.eu">>
}, Req0).

请注意，头名称必须始终是小写的二进制数据。

当您还需要设置响应体时，请使用 cowboy_req:reply/4：

Req = cowboy_req:reply(200, #{
    <<"content-type">> => <<"text/plain">>
}, "Hello world!", Req0).

当响应有体时，您应该始终设置 content-type 头。然而，不需要设置 content-length 头；Cowboy 会自动处理。

响应体和头部值必须是二进制数据或 iolist。iolist 是一个包含二进制数据、字符、字符串或其他 iolist 的列表。这允许你从不同的部分构建响应，而无需进行任何连接操作：

Title = "Hello world!",
Body = <<"Hats off!">>,
Req = cowboy_req:reply(200, #{
    <<"content-type">> => <<"text/html">>
}, ["<html><head><title>", Title, "</title></head>",
    "<body><p>", Body, "</p></body></html>"], Req0).

这种构建响应的方法比连接更高效。幕后，列表中的每个元素只是一个指针，这些指针在写入套接字时直接使用。

流式回复

Cowboy 提供了两个用于启动响应的函数，以及一个用于流式传输响应体的函数。Cowboy 会向响应添加任何必要的头部。

当您只需要设置状态码时，请使用 cowboy_req:stream_reply/2：

Req = cowboy_req:stream_reply(200, Req0),

cowboy_req:stream_body("Hello...", nofin, Req),
cowboy_req:stream_body("chunked...", nofin, Req),
cowboy_req:stream_body("world!!", fin, Req).

cowboy_req:stream_body/3 的第二个参数表示此数据是否终止主体。使用 fin 作为最终标志，否则使用 nofin。

该片段没有设置内容类型头。这不被推荐。所有带有主体的响应都应该设置内容类型。头可以在之前设置，或者使用 cowboy_req:stream_reply/3 设置：

Req = cowboy_req:stream_reply(200, #{
    <<"content-type">> => <<"text/html">>
}, Req0),

cowboy_req:stream_body("<html><head>Hello world!</head>", nofin, Req),
cowboy_req:stream_body("<body><p>Hats off!</p></body></html>", fin, Req).

HTTP 提供了几种不同的方式来流式传输响应主体。Cowboy 将根据 HTTP 版本和请求及响应头选择最合适的流式传输方式。

虽然并非强制要求，但如果提前知道响应内容的长度，建议设置 content-length 头。这将确保选择最佳的响应方法，并帮助客户端了解何时接收到完整的响应。

Cowboy 还提供了一个发送响应尾部的功能。响应尾部在语义上与您在响应中发送的头信息等同，只是它们在响应结束时发送。这特别适用于在响应主体完全生成后才能生成的信息。

尾部字段必须在尾部头中列出。任何未列出的字段可能会被客户端或中间件丢弃。

Req = cowboy_req:stream_reply(200, #{
    <<"content-type">> => <<"text/html">>,
    <<"trailer">> => <<"expires, content-md5">>
}, Req0),

cowboy_req:stream_body("<html><head>Hello world!</head>", nofin, Req),
cowboy_req:stream_body("<body><p>Hats off!</p></body></html>", nofin, Req),

cowboy_req:stream_trailers(#{
    <<"expires">> => <<"Sun, 10 Dec 2017 19:13:47 GMT">>,
    <<"content-md5">> => <<"c6081d20ff41a42ce17048ed1c0345e2">>
}, Req).

流结束于尾部。发送尾部后将无法再发送数据。在流式传输主体时，设置 fin 标志后，无法再发送尾部。

预设响应头

Cowboy 提供了设置响应头而不立即发送的功能。这些头信息存储在 Req 对象中，并在调用回复函数时作为响应的一部分发送。

要设置响应头：

Req = cowboy_req:set_resp_header(<<"allow">>, "GET", Req0).

头名称必须是小写的二进制字符串。

不要使用此功能设置 cookies。请参阅 cookies 章节以获取更多信息。

要检查响应头是否已设置：

cowboy_req:has_resp_header(<<"allow">>, Req).

如果设置了头信息，则返回 true，否则返回 false。

要删除之前设置的响应头：

Req = cowboy_req:delete_resp_header(<<"allow">>, Req0).

覆盖头部

Cowboy 提供了设置响应头和正文的不同方式，因此在设置头信息时可能会发生冲突，因此了解在头信息被设置两次时会发生什么是很重要的。

Headers 来自五个不同的来源：

  • 协议特定的头部字段（例如 HTTP/1.1 的 connection 头部）
  • 其他必需的头部信息（例如日期头部）
  • 预设头
  • 回复函数的头信息
  • Set-cookie 头

牛仔不允许覆盖协议特定的头信息。

Set-cookie 头会在发送响应之前始终被添加到头列表的末尾。

给 reply 函数的头信息会始终覆盖预设头和必需头。如果在两种或三种中发现了相同的头，则会选择 reply 函数中的头信息并丢弃其他头信息。

同样，预设头会始终覆盖必需头。

例如，查看以下代码片段。Cowboy 默认会发送 server 头，值为"Cowboy"。我们可以覆盖它：

Req = cowboy_req:reply(200, #{
    <<"server">> => <<"yaws">>
}, Req0).

预设响应主体

Cowboy 提供了设置响应体而不立即发送的功能。它存储在 Req 对象中，并在调用回复函数时发送。

要设置响应体：

Req = cowboy_req:set_resp_body("Hello world!", Req0).

要检查是否已经设置了响应体：

cowboy_req:has_resp_body(Req).

如果设置了非空的响应体，则返回 true，否则返回 false。

The preset response body is only sent if the reply function used is cowboy_req:reply/2 or cowboy_req:reply/3.

发送文件

Cowboy provides a shortcut for sending files. When using cowboy_req:reply/4, or when presetting the response header, you can give a sendfile tuple to Cowboy:

{sendfile, Offset, Length, Filename}

Depending on the values for Offset or Length, the entire file may be sent, or just a part of it.

The length is required even for sending the entire file. Cowboy sends it in the content-length header.

发送文件的同时回复：

Req = cowboy_req:reply(200, #{
    <<"content-type">> => "image/png"
}, {sendfile, 0, 12345, "path/to/logo.png"}, Req0).

信息性回复

Cowboy 允许你发送信息性的响应。

信息响应的状态码在100到199之间。在发送正确的响应之前可以发送任何数字。发送信息响应不会改变正确响应的行为，客户端应忽略他们不理解的信息响应。

以下片段发送一个103信息响应，并附带一些预期在最终响应中出现的头部。

Req = cowboy_req:inform(103, #{
    <<"link">> => <<"</style.css>; rel=preload; as=style, </script.js>; rel=preload; as=script">>
}, Req0).

Push

HTTP/2 协议引入了推送与响应一起发送的相关资源的能力。Cowboy 提供了两个函数用于此目的：cowboy_req:push/3,4。

推送仅适用于 HTTP/2。如果协议不支持，Cowboy 会自动忽略推送请求。

在调用任何回复函数之前，必须调用 push 函数。否则会导致程序崩溃。

要推送一个资源，您需要提供与客户端执行请求时相同的信息。这包括 HTTP 方法、URI 和任何必要的请求头。

Cowboy 默认只需要提供资源的路径和请求头。URI 的其余部分将从当前请求中获取（排除查询字符串，设置为空），默认方法为 GET。

以下片段推送了一个链接到响应中的 CSS 文件：

cowboy_req:push("/static/style.css", #{
    <<"accept">> => <<"text/css">>
}, Req0),
Req = cowboy_req:reply(200, #{
    <<"content-type">> => <<"text/html">>
}, ["<html><head><title>My web page</title>",
    "<link rel='stylesheet' type='text/css' href='/static/style.css'>",
    "<body><p>Welcome to Erlang!</p></body></html>"], Req0).

要覆盖方法、方案、主机、端口或查询字符串，只需传递第四个参数即可。以下代码片段使用了不同的主机名：

cowboy_req:push("/static/style.css", #{
    <<"accept">> => <<"text/css">>
}, #{host => <<"cdn.example.org">>}, Req),

推送的资源不必是文件。只要推送请求是可缓存的、安全的且不包含主体，资源就可以被推送。

在幕后，Cowboy 处理推送请求与普通请求相同：会创建一个不同的进程，最终将响应发送给客户端。
