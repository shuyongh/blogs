实例

% hello world program
-module(helloworld). 
-export([start/0]). 

start() -> 
   io:fwrite("Hello, world!\n").
  • ％ 符号用于向程序添加注释。
  • module 语句就像在任何编程语言中添加命令空间一样。在这里，我们要提到的是，这段代码将是一个名为 helloworld 的模块的一部分。
  • 使用 export 函数可以使用程序中定义的任何函数。我们正在定义一个名为 start 的函数，为了使用 start 函数，我们必须使用 export 语句。/0 表示我们的函数‘start’ 接受 0 参数.
  • 我们最终定义了 start 函数。这里我们使用另一个名为 io 的模块，它在 Erlang 中具有所有必需的输入输出函数。我们使用 fwrite 函数将“Hello World”输出到控制台。

声明形式

Erlang 语言中使用了不同的符号

  • 连字符(–)通常与模块，导入和导出语句一起使用。连字符用于为每个语句赋予相应的含义。因此，Hello world 程序的示例显示在以下程序中-
-module(helloworld).
-export([start/0]).

每个语句都用点(.)符号定界。Erlang 中的每个语句都需要以该定界符结尾。Hello world 程序的示例如下例所示：

io:fwrite("Hello, world!\n").
  • 斜杠 (/) 符号与函数一起使用，以定义函数接受的参数数量。
-export([start/0]).

模块

在 Erlang 中，所有代码都分为模块。模块由一系列属性和函数声明组成。就像其他编程语言中的名称空间的概念一样，该名称空间用于逻辑上分离不同的代码单元。

  • 定义
    • 该 ModuleName 需求是相同的文件名减去扩展 .erl。否则，代码加载将无法按预期进行。
-module(ModuleName)

导入声明

在 Erlang 中，如果要使用现有 Erlang 模块的功能，则可以使用 import 语句

-import (modulename, [functionname/parameter]).
  • 模块名 − 这是需要导入的模块的名称。
  • 函数名称/参数 − 模块中需要导入的功能。

更改编写 hello world 程序以使用 import 语句的方式

% hello world program
-module(helloworld).
-import(io,[fwrite/1]).
-export([start/0]).

start() ->
   fwrite("Hello, world!\n").

在上面的代码中，我们使用 import 关键字导入库“ io”，尤其是 fwrite 函数。因此，现在无论何时调用 fwrite 函数，都不必在任何地方都提及 io 模块名称。

常用符号

逗号和分号

关键点分号（;）逗号（,）
用途1. 分隔函数中的多个子句（模式匹配）。1. 分隔函数体内的多个表达式。
2. 分隔控制结构（如 case, if, receive）中的多个分支。2. 分隔元组或列表中的多个元素。
位置通常用于函数定义和控制结构的分支中。通常用于函数体内、元组、列表等地方。
示例max(A, B) when A > B -> A; max(A, B) -> B.C = A + B, D = A * B, C + D.
作用用来分隔不同的子句或者分支。用来分隔同一作用域内的多个表达式或元素。

逗号

逗号用于 分隔同一函数内部的多个表达式，即在同一作用域内分隔不同的表达式或语句。它通常出现在 函数体 中或者在 元组 和 列表 中。

分号

在函数中分隔多个子句：当一个函数有多个子句（模式匹配）时，每个子句之间使用 ； 进行分隔。每个子句通常由 -> 和匹配的条件部分组成。

在控制结构中分隔多个表达式：比如在 case、if、receive 等结构中，每个分支的代码块也使用 ； 分隔。

管道符

在列表模式匹配中的使用

| 用于模式匹配中的列表操作，通常用于将列表的第一个元素和剩余部分分开。它可以用于获取列表的头部和尾部。

[Head | Tail]

  • Head：列表的第一个元素。

  • Tail：列表的其余部分（尾部），通常是一个列表。

在记录定义中的使用管道符

通过 | 来扩展字段类型（类似于在列表操作中的尾部扩展）

-record(person, {name, age = 18, city = "Unknown"}).
-record(employee, {salary = 50000} | #person{}).  % 使用 | 扩展 person 记录

这里的 | 用于扩展记录，将 #person{} 记录类型添加到 employee 类型中。这意味着，employee 记录除了包含 salary 字段外，还会自动包含 person 记录中的所有字段（name、age 和 city）。

连字符

连字符 (-) 用于声明和定义一些特殊的概念、指令和类型，这些概念在 Erlang 语言中具有重要作用。

  • 模块声明：-module(Name). 用于声明模块。

  • 函数导出：-export([Function/Arity]). 用于声明模块对外暴露的函数。

  • 类型声明：-type Name() :: Type. 用于声明自定义类型。

  • 函数规格：-spec Function(Arg1Type, Arg2Type) -> ReturnType. 用于声明函数的输入输出类型。

  • 记录声明：-record(Name, {Field1, Field2}). 用于声明记录类型。

  • 编译选项：-compile([Option]). 用于设置编译选项。

  • 行为声明：-behaviour(BehaviourName). 用于声明模块实现某种行为。

  • 编译宏：-define(MacroName, Value). 用于声明编译时常量。

模块声明

模块是 Erlang 程序中的基本组织单元。每个模块通常由一个 .erl 文件表示，模块的定义包括函数、类型、记录等。模块提供封装，支持命名空间，从而帮助程序员更好地组织代码。

  • 模块包含一组相关的函数。

  • 每个模块有一个唯一的名字，通常文件名与模块名相同。

连字符的作用：

  • -module 用来声明模块的名称。
-module(math).

函数导出

Erlang 中的函数默认是 私有的，只有在使用 -export 声明后才会对外公开，使得其他模块可以调用

  • 导出函数 是指可以被外部模块调用的函数。

连字符的作用：

  • -export 用于声明哪些函数可以被其他模块访问。
-export([add/2, subtract/2]).

导入声明

在 Erlang 中，如果要使用现有 Erlang 模块的功能，则可以使用 import 语句

-import (modulename, [functionname/parameter]).
  • 模块名 − 这是需要导入的模块的名称。
  • 函数名称/参数 − 模块中需要导入的功能。

类型导出

export_type 用于标记哪些类型定义是模块外部可访问的。

-export_type([Type1, Type2, ...]).

类型声明

Erlang 支持基本的 类型注解，但并不像某些静态类型语言那样强制执行。-type 声明用于定义用户自定义的类型，这对于文档、代码分析和静态检查（如 Dialyzer）非常有用。

  • 类型 描述了变量或表达式的预期数据结构，可以是基本类型如整数、浮点数，也可以是复杂的用户自定义类型。
语法
  • -type 用来定义自定义类型。
-type TypeName() :: Type.

  • TypeName：类型别名的名称。通常使用大写字母开头。

  • Type：实际的类型定义。可以是任何有效的 Erlang 类型（如原子、整数、列表、元组等）。

基本使用
基本类型别名
-type age() :: integer().
-type name() :: string().

age() 表示整数类型，name() 表示字符串类型。

这样，你可以在函数中使用 age() 和 name() 来表示这些类型，而不需要每次都写出 integer() 和 string()。

元组类型别名
-type person() :: {name(), age()}.

person() 是一个元组类型，包含两个元素，分别是 name() 和 age() 类型。

列表类型别名
-type name_list() :: [name()].

name_list() 是一个包含 name() 类型元素的列表。

记录类型别名
-type person_record() :: #person{name :: string(), age :: integer()}.

person_record() 是一个包含 name 和 age 字段的记录类型的别名。

函数类型别名
-type add_fun() :: (integer(), integer() -> integer()).

add_fun() 表示一个接受两个整数并返回一个整数的函数类型。

函数规格

函数规格用于描述函数的类型签名，特别是在函数的输入、输出及参数类型方面进行说明。-spec 可以帮助生成更具可读性的文档，并且可以用于静态分析工具（如 Dialyzer）来验证代码的正确性。

  • 函数规格 用来描述函数的参数类型和返回值类型。

  • -spec 可以帮助静态分析工具（如 Dialyzer）检测潜在的类型错误。

连字符的作用：

  • -spec 用来声明函数的类型和签名。
-spec add(integer(), integer()) -> integer().

常见的类型和其描述：

  • integer(): 所有整数（包括正整数、负整数和零）。
  • float(): 所有浮点数。
  • atom(): 所有原子（例如：ok、error）。
  • string(): 字符串，实际上是一个字符列表（list(char())）。
  • list(T): 一个元素类型为 T 的列表。
  • tuple(T1, T2, ...): 一个元组，包含多个元素，类型为 T1, T2, ...。
  • pid(): 进程标识符。
  • port(): 外部端口。
  • reference(): 引用类型。
  • term() ： 根类型，表示所有类型。
  • non_neg_integer() ：是 Erlang 的标准类型，表示一个大于或等于 0 的整数。通常用来表示子列表的长度。
  • pos_integer() ：是 Erlang 的标准类型，它表示大于 0 的整数，通常用来表示列表的索引位置。
  • iodata(): 可以包含 二进制数据、字符列表 或者它们的组合的结构。

记录声明

记录（record）是 Erlang 中用于表示结构化数据的方式。记录 类似于其他语言中的结构体（struct），它允许将多个数据项（字段）组合在一起。

  • 记录 是一个数据结构，由多个命名的字段组成。

  • 可以通过字段名访问记录中的数据。

  • 记录是 不可变 的数据结构。

  • 记录字段名是 预定义的，通过 -record 来指定，字段的名字在定义时固定。

  • 字段可以在定义时 指定默认值，如果创建记录时没有提供该字段值，则使用默认值。

基本语法
  • -record 用来定义一个新的记录类型。
-record(Name, {Field1, Field2, ..., FieldN}).

  • Name：记录的名称（即类型名称），通常是原子（atom）。

  • Field1, Field2, ..., FieldN：记录的字段名，字段名之间用逗号分隔。

    • 字段可以是任意 Erlang 类型（例如原子、整数、字符串、列表等），甚至是其他记录类型。
操作
使用

使用 -record 来定义记录

使用 # 运算符来创建记录的实例

-record(person, {name = "Unknown", age = 0, city = "Unknown"}),
P1 = #person{ age = 30},
io:fwrite("~p~n",[P1#persion.age]);
io:fwrite("~p~n",[P1]);
% 输出  30
%		#person{name = "Unknown", age = 25, city = "Unknown"}
通过管道符来扩展字段类型

通过 | 来扩展字段类型（类似于在列表操作中的尾部扩展）

-record(person, {name, age = 18, city = "Unknown"}).
-record(employee, {salary = 50000} | #person{}).  % 使用 | 扩展 person 记录

这里的 | 用于扩展记录，将 #person{} 记录类型添加到 employee 类型中。这意味着，employee 记录除了包含 salary 字段外，还会自动包含 person 记录中的所有字段（name、age 和 city）。

编译选项

编译选项用于控制 Erlang 编译器的行为，像是优化级别、是否生成调试信息、是否启用警告等。

  • 编译选项 控制 Erlang 编译器如何处理源代码。

连字符的作用：

  • -compile 用于设置编译选项。
% 告诉编译器将当前模块中的所有函数自动导出，而不需要显式地列出每个函数的导出
-compile([export_all]).

行为

行为是 Erlang 中的一种机制，用于通过约定定义模块之间的接口。行为类似于其他语言中的接口或抽象类。通过行为，模块可以实现特定的约定，如进程管理、事件驱动等。

  • 行为 定义了一组标准的函数签名，供模块遵循。

连字符的作用：

  • -behaviour 用于声明模块实现某种行为。
定义行为

要求实现模块必须提供两个回调函数：init/1 和 handle/2。

-module(my_behaviour).
%% 标记为可选回调函数。
%% 如果在实现 my_behaviour 行为时没有提供这些函数的实现，Erlang 编译器也不会报错。
-optional_callbacks([handle_info/2]).


-callback init(Args :: term()) -> {ok, State :: term()} | {error, Reason :: term()}.
-callback handle(Request :: term(), State :: term()) -> {reply, Reply :: term(), NewState :: term()}.
-callback handle_info(Info :: timeout | term(), State :: term()) ->
    {noreply, NewState :: term()} |
    {noreply, NewState :: term(), timeout() | hibernate | {continue, term()}} |
    {stop, Reason :: term(), NewState :: term()}.
实现行为

任何模块要实现这个行为，必须提供 init/1 和 handle/2 函数，handle_info/2可选择性实现

-module(my_module).
-behaviour(my_behaviour).

-export([init/1, handle/2]).

init(Args) ->
    {ok, Args}.  % 初始化状态

handle(Request, State) ->
    {reply, Request, State}.  % 处理请求并返回结果

编译宏

Erlang 允许在编译时使用宏来简化常用的代码模式，类似于其他语言中的 #define 宏。

  • 编译宏 可以用于在编译时定义常量或替换代码片段。

连字符的作用：

  • -define 用于声明宏。
-define(MAX_SIZE, 100).

异常处理

在 Erlang 中，连字符也用于声明异常信息和错误处理的相关信息。

-throw(error).

::

:: 是一种用作 类型约束 的操作符，用于指定变量的类型或为函数的参数设定类型要求。它通常出现在模式匹配中或在定义函数时标注变量类型。通过使用 ::，你可以确保变量或函数的参数符合预期的类型。

语法
Var :: Type

  • Var 是你要进行类型约束的变量。

  • Type 是你希望该变量匹配的类型。

示例
简单类型约束
X :: integer.  % X必须是一个整数

在上面的代码中，X 变量被约束为必须是一个整数。虽然 Erlang 是动态类型语言，但你可以使用 :: 来强制类型匹配，以便在运行时捕获类型不匹配的错误。

在模式匹配中的应用
is_integer(X) when X :: integer -> true;
is_integer(_) -> false.

在上面的代码中，函数 is_integer/1 检查其参数 X 是否是整数。如果 X 是整数，则返回 true，否则返回 false。

类型注解与函数签名
-spec add(integer(), integer()) -> integer().
add(X, Y) when X :: integer, Y :: integer -> X + Y.

使用 -spec 来给函数 add/2 添加类型注解，表示该函数接受两个整数参数并返回一个整数。这有助于 Dialyzer 静态检查并捕获类型不匹配的错误。

:=

用于更新**映射（map）**中某个键的值的操作符

用于创建一个新的映射，在新映射中，某个键的值被更新，而其他键值对保持不变。

1> Map = #{name => "Alice", age => 30}.
#{name => "Alice", age => 30}
2> UpdatedMap = Map#{age := 31}.
#{name => "Alice", age => 31}

预处理器

在编译 Erlang 模块之前，Erlang 预处理器会自动处理该模块。预处理器扩展源文件中可能存在的任何宏，并插入任何必要的包含文件。

通常，您不需要查看预处理器的输出，但在特殊情况下(例如，调试一个错误的宏时)，您可能希望保存预处理器的输出。

查看对模块 some_module 进行预处理的结果。erl 给出操作系统的 shell 命令。

erlc -P some_module.erl

这会生成一个名为 some_module.P 的清单文件

动态代码载入

动态代码载入是内建于 Erlang 核心的最惊人特性之一

每当调用 someModule:someFunction(...) 时，调用的总是最新版模块里的最新版函数，哪怕当代码在模块里运行时重新编译了该模块也是如此
