接口技术

约 2756 字大约 9 分钟

2025-03-22

在 Erlang 中，"接口技术" 通常是指如何使 Erlang 程序与外部世界进行交互，包括与其他编程语言、外部服务、硬件设备等进行通信

Erlang 运行时系统内置了两种互操作机制，即 分布式 Erlang、端口 和 NIF。端口的一种变体是链接式驱动。

问题引入

一个常见的互操作性场景是，当你想在你的 Erlang 程序中整合一段解决复杂问题的代码时。例如，假设你有以下 C 函数，你想从 Erlang 中调用它们

/* complex.c */

int foo(int x) {
  return x+1;
}

int bar(int y) {
  return y*2;
}

为了便于阅读，这些函数被刻意保持得尽可能简单。

从 Erlang 的角度来看，最好能够调用 foo 和 bar，而无需关心它们是 C 函数

% Erlang code
...
Res = complex:foo(X),
...

在这里，与 C 的通信隐藏在 complex.erl 的实现中。在接下来的章节中，将展示如何使用不同的互操作性机制来实现此模块。

端口

提示

从 Erlang 的角度来看，端口提供了与外部世界通信的基本机制。端口为外部程序提供了一个面向字节的接口。创建端口后，Erlang 可以通过发送和接收 字节列表 或 二进制数据（而不是 Erlang 项）与它通信。

链接式驱动：外部程序驻留在与 Erlang 运行时系统不同的操作系统进程中。在某些情况下，这是不可接受的。例如，考虑具有非常严格时间要求的驱动程序。因此，可以按照一定的原则用 C 编写程序，并将其动态链接到 Erlang 运行时系统。这被称为 链接式驱动。

端口和进程的关系：Erlang 中的端口本质上是一个特殊的进程，但它不由 Erlang 虚拟机（BEAM）调度，它通过消息与 Erlang 进程通信。端口进程通常用于与外部程序进行双向通信。

**何时使用 ： 端口可用于 Erlang 程序和其他程序在同一台计算机上运行的各种互操作情况。

常用方法

方法说明示例
open_port(Args, Options)打开一个端口，启动外部程序，并返回一个端口标识符
port_command(Port, Data)向端口发送数据
port_info(Port)获取有关端口的信息, 接收端口数据
flush()接收从端口发送到当前进程的所有消息
port_close(Port)关闭端口

在 Erlang 中，端口（Port）是与外部程序交互的机制，使用端口时会涉及多个常用的函数和参数。以下是对 Erlang 中端口相关的常用函数及其参数的详细介绍。

1. open_port/2

open_port/2 用于打开一个端口，启动外部程序，并返回一个端口标识符 Port。这个端口标识符用于与外部程序进行交互。

函数签名

-spec open_port(PortName, PortSettings) -> port()
                   when
                       PortName ::
                           {spawn, Command :: string() | binary()} |
                           {spawn_driver, Command :: string() | binary()} |
                           {spawn_executable, FileName :: file:name_all()} |
                           {fd, In :: non_neg_integer(), Out :: non_neg_integer()},
                       PortSettings :: [Opt],
                       Opt ::
                           {packet, N :: 1 | 2 | 4} |
                           stream |
                           {line, L :: non_neg_integer()} |
                           {cd, Dir :: string() | binary()} |
                           {env,
                            Env :: [{Name :: os:env_var_name(), Val :: os:env_var_value() | [] | false}]} |
                           {args, [string() | binary()]} |
                           {arg0, string() | binary()} |
                           exit_status | use_stdio | nouse_stdio | stderr_to_stdout | in | out |
                           binary | eof |
                           {parallelism, Boolean :: boolean()} |
                           hide |
                           {busy_limits_port, {non_neg_integer(), non_neg_integer()} | disabled} |
                           {busy_limits_msgq, {non_neg_integer(), non_neg_integer()} | disabled}.

参数

PortName 用来指定外部进程或设备的启动方式。它可以是以下几种格式：

  • {spawn, Command :: string() | binary()}

    • 描述：通过 spawn 启动外部命令或脚本。

    • 参数：Command 是要执行的命令，可以是字符串或二进制格式。

  • {spawn_driver, Command :: string() | binary()}

    • 描述：启动外部命令，作为一个 Erlang 驱动程序来处理 I/O。

    • 参数：Command 是要启动的驱动命令。

  • {spawn_executable, FileName :: file:name_all()}

    • 描述：启动外部可执行文件。

    • 参数：FileName 是文件路径，指定要执行的可执行文件。

  • {fd, In :: non_neg_integer(), Out :: non_neg_integer()}

    • 描述：通过文件描述符（FD）创建端口。这用于直接与文件或管道进行通信。

    • 参数：In 和 Out 是文件描述符，分别表示输入和输出通道。

PortSettings 是一组选项，用于配置如何与端口进行交互。它是一个包含多个配置项的列表。

  • {packet, N :: 1 | 2 | 4}

    • 描述：设置如何通过端口发送数据包。

    • 参数：N 是数据包的大小，允许值为 1、2 或 4。

  • stream

    • 描述：设置端口为流模式，这意味着数据将按字节流的方式发送和接收。

  • {line, L :: non_neg_integer()}

    • 描述：设置端口每次接收的最大字节数 L。通常用于处理文本流数据时，按行读取。

  • {cd, Dir :: string() | binary()}

    • 描述：指定端口启动时的工作目录（current directory）。

    • 参数：Dir 是一个字符串或二进制表示的目录路径。

  • {env, Env :: [{Name :: os:env_var_name(), Val :: os:env_var_value() | [] | false}]}

    • 描述：为外部进程设置环境变量。

    • 参数：Env 是一个列表，包含环境变量的名称和值。

  • {args, [string() | binary()]}

    • 描述：指定要传递给外部命令的参数。

    • 参数：args 是一个字符串或二进制列表，表示要传递的命令行参数。

  • {arg0, string() | binary()}

    • 描述：指定外部命令的第一个参数（通常是程序名称）。

    • 参数：arg0 是一个字符串或二进制，表示第一个参数。

  • exit_status

    • 描述：表示是否捕获外部进程的退出状态。

  • use_stdio

    • 描述：表示标准输入/输出应该用于与端口的通信。

  • nouse_stdio

    • 描述：表示不要使用标准输入/输出与端口通信。

  • stderr_to_stdout

    • 描述：将标准错误（stderr）重定向到标准输出（stdout）。

  • in | out

    • 描述：分别指定端口的输入或输出通道。

  • binary

    • 描述：表示数据将以二进制格式进行传输。

  • eof

    • 描述：表示数据流的结尾，通常用于处理文本文件或流。

  • {parallelism, Boolean :: boolean()}

    • 描述：表示是否启用并行化处理。

    • 参数：Boolean 可以是 true 或 false。

  • hide

    • 描述：表示端口应隐藏其输出。

  • {busy_limits_port, {non_neg_integer(), non_neg_integer()} | disabled}

    • 描述：表示设置端口的忙碌限制（如队列长度等）。disabled 表示没有限制。

  • {busy_limits_msgq, {non_neg_integer(), non_neg_integer()} | disabled}

    • 描述：表示设置消息队列的忙碌限制。disabled 表示没有限制。

返回值

open_port/2 函数返回一个 port() 类型，表示与外部进程或设备的通信端口。你可以通过这个端口与外部进程进行消息交换。

3. port_info/1

port_info/1 用于获取关于端口的详细信息。它返回一个包含端口状态的元组或列表，这有助于检查端口的运行状态或调试。

函数签名：

port_info(Port) -> Info

参数：

  • Port: 端口标识符，通常是由 open_port/2 返回的值。

返回值：

port_info/1 返回的值是一个包含有关端口的详细信息的元组。常见的信息有：

  • {name, Program}: 端口对应的外部程序。
  • {status, Status}: 端口的当前状态（例如，{open, Program} 表示端口正在打开，或者 {closed} 表示端口已关闭）。
  • {exit_status, Code}: 外部程序退出时的状态码（如果适用）。
  • {data, Data}: 端口读取到的数据。

flush/0

flush/0 是用于接收从端口发送到当前进程的所有消息的函数。这在处理来自多个端口的消息时非常有用。

函数签名：

flush() -> Msgs

参数：

没有参数。

返回值：

flush/0 会返回当前进程消息队列中的所有消息。

示例

Erlang 程序

Erlang 和 C 之间的所有通信都必须通过创建端口来建立。创建端口的 Erlang 进程被称为端口的 连接进程。所有进出端口的通信都必须经过连接进程。如果连接进程终止，端口也会终止（如果外部程序编写正确，外部程序也会终止）。

-module(complex1).
-export([start/1, stop/0, init/1]).
-export([foo/1, bar/1]).

start(ExtPrg) ->
    spawn(?MODULE, init, [ExtPrg]).
stop() ->
    complex ! stop.


% 实现 complex1:foo/1 和 complex1:bar/1。
% 两者都向 complex 进程发送消息并接收以下回复
foo(X) ->
    call_port({foo, X}).
bar(Y) ->
    call_port({bar, Y}).

call_port(Msg) ->
    complex ! {call, self(), Msg},
    receive
        {complex, Result} ->
            Result
    end.

% complex 进程执行以下操作
    % 1.将消息编码为字节序列。
    % 2.将其发送到端口。
    % 3.等待回复。
    % 4.解码回复。
    % 5.将其发送回调用者
loop(Port) ->
    receive
        {call, Caller, Msg} ->
            Port ! {self(), {command, encode(Msg)}},
            receive
                {Port, {data, Data}} ->
                    Caller ! {complex, decode(Data)}
            end,
            loop(Port);
        stop ->
            Port ! {self(), close},
            receive
                {Port, closed} ->
                    exit(normal)
            end;
        {'EXIT', Port, Reason} ->
            exit(port_terminated)
    end.

%假设 C 函数的参数和结果都小于 256，则采用简单的编码/解码方案。
%在此方案中，foo 用字节 1 表示，bar 用 2 表示，参数/结果也用单个字节表示
encode({foo, X}) -> [1, X];
encode({bar, Y}) -> [2, Y].

% 创建端口
init(ExtPrg) ->
    register(complex, self()),
    %设置为捕获退出，这使得能够检测外部程序的失败
    process_flag(trap_exit, true),
    %字符串 ExtPrg 是外部程序的名称，包括任何命令行参数
    %{packet,2}。此选项表示使用 2 字节的长度指示符来简化 C 和 Erlang 之间的通信。
    %Erlang 端口会自动添加长度指示符，但必须在外部 C 程序中显式完成此操作。
    Port = open_port({spawn, ExtPrg}, [{packet, 2}]),
    loop(Port).




decode([Int]) -> Int.

C 程序

在 C 端，有必要编写函数来从/向 Erlang 接收和发送带有 2 字节长度指示符的数据。默认情况下，C 程序应从标准输入（文件描述符 0）读取并写入标准输出（文件描述符 1）

  • 请注意，stdin 和 stdout 用于缓冲输入/输出，不能用于与 Erlang 的通信。
/* erl_comm.c */

#include <stdio.h>
#include <unistd.h>

typedef unsigned char byte;

int read_exact(byte *buf, int len)
{
  int i, got=0;

  do {
      if ((i = read(0, buf+got, len-got)) <= 0){
          return(i);
      }
    got += i;
  } while (got<len);

  return(len);
}

int write_exact(byte *buf, int len)
{
  int i, wrote = 0;

  do {
    if ((i = write(1, buf+wrote, len-wrote)) <= 0)
      return (i);
    wrote += i;
  } while (wrote<len);

  return (len);
}

int read_cmd(byte *buf)
{
  int len;

  if (read_exact(buf, 2) != 2)
    return(-1);
  len = (buf[0] << 8) | buf[1];
  return read_exact(buf, len);
}

int write_cmd(byte *buf, int len)
{
  byte li;

  li = (len >> 8) & 0xff;
  write_exact(&li, 1);

  li = len & 0xff;
  write_exact(&li, 1);

  return write_exact(buf, len);
}
  • 在 main 函数中，C 程序应监听来自 Erlang 的消息，并根据选定的编码/解码方案，使用第一个字节来确定要调用的函数，第二个字节作为函数的参数。然后将调用函数的结果发送回 Erlang。
/* port.c */

typedef unsigned char byte;

int main() {
  int fn, arg, res;
  byte buf[100];

  while (read_cmd(buf) > 0) {
    fn = buf[0];
    arg = buf[1];

    if (fn == 1) {
      res = foo(arg);
    } else if (fn == 2) {
      res = bar(arg);
    }

    buf[0] = res;
    write_cmd(buf, 1);
  }
}
