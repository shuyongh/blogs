约 6912 字大约 23 分钟

ETS 和 DETS

ets 和 dets 是两个系统模块，可以用来高效存储海量的 Erlang 数据。ETS 是 Erlang Term Storage（Erlang 数据存储）的缩写，DETS 则是 Disk ETS（磁盘 ETS）的缩写。

ETS 和 DETS 执行的任务基本相同：它们提供大型的键-值查询表。ETS常驻内存，DETS则常驻磁盘。ETS 是相当高效的：可以用它存储海量的数据（只要有足够的内存），执行查找的时间也是恒定的（在某些情况下是对数时间）。DETS 提供了几乎和 ETS 一样的接口，但它会把表保存在磁盘上。因为 DETS 使用磁盘存储，所以它远远慢于 ETS，但是运行时的内存占用也会小很多。另外，ETS和DETS表可以被多个进程共享，这就让跨进程的公共数据访问变得非常高效。ETS 和 DETS 表是把键和值关联到一起的数据结构。最常用的表操作是插入和查找。ETS或DETS表其实就是Erlang元组的集合。

ETS表里的数据保存在内存里，它们是易失的。当 ETS 表被丢弃或者控制它的 Erlang 进程终止时，这些数据就会被删除。保存在DETS表里的数据是非易失的，即使整个系统崩溃也能留存下来。DETS 表在打开时会进行一致性检查，如果发现有损坏，系统就会尝试修复它（这可能会花费很长时间，因为表里的所有数据都要检查）。这应该能恢复表里的所有数据，但如果表里的最后一项是在系统崩溃时生成的，就可能会丢失。

ETS 表广泛应用于那些必须以高效方式操作大量数据的应用程序，以及用非破坏性赋值和“纯”Erlang 数据结构编程的开销过大之时。

ETS 表看上去像是用 Erlang 实现的，但事实上它们是在底层的运行时系统里实现的，有着不同于普通 Erlang 对象的性能特点。特别需要指出的是，ETS表没有垃圾收集机制，这就意味着即使ETS表极其巨大也不会有垃圾收集的负担。不过，创建或访问 ETS 对象仍然会带来少许开销。

表的类型

ETS 和 DETS 表保存的是元组。元组里的某一个元素（默认是第一个）被称为该表的键。通过键来向表里插入和提取元组。当我们向表里插入一个元组时会发生什么，取决于表的类型和键的值。

  • 异键表（set）: 各个元组里的键都必须是独一无二的。
    • 有序异键（ordered set）：元组会被排序。
  • 同键表（bag）: 它们允许多个元素拥有相同的键。
    • 副本同键（duplicate bag）：可以存在多个相同的元组。

ETS

创建表

%% ets表的数据结构为 [{Key, Value1,Value2....},...]
ets:new(Name, Options) -> Table

  • Name：ETS 表的名称，通常是一个原子（atom），用于标识表。这个名称对于进程间共享表来说是必需的。

  • Options：一个包含表行为的选项的列表。选项定义了表的特性和行为。

以下是 ets:new/2 函数中常见的选项及其作用：

选项类型说明
setatom表示创建一个 set 类型的表，表中没有重复的键（即主键唯一）。
ordered_setatom创建一个按键有序的 ordered_set 类型表，表中没有重复的键。
bagatom创建一个 bag 类型的表，允许同一键有多个值。
duplicate_bagatom创建一个 duplicate_bag 类型的表，允许同一键有多个相同的值。
publicatom表示该表是公开的，可以被所有进程访问。
protectedatom表示该表是受保护的，只能由创建它的进程访问。
privateatom表示该表是私有的，只能由创建它的进程访问，其他进程无法访问。
write_concurrencyatom允许多个进程同时写入表。
read_concurrencyatom允许多个进程同时读取表。
named_tableatom使 ETS 表成为命名表，允许其他进程通过 ets:lookup/2 等函数查找该表。
compressedatom启用表内容的压缩。

删除表

删除一个 ETS（Erlang Term Storage）表。它通过指定表的标识符（TableId）来删除整个表，而不仅仅是删除表中的单个条目。

ets:delete(TableId) -> ok | {error, Reason}

参数

  • TableId：要删除的 ETS 表的标识符。通常，TableId 是在创建表时返回的值。

返回值

  • ok：表示操作成功，即表已成功删除。
  • {error, Reason}：如果删除失败，返回一个错误元组，Reason 表示错误原因。

清空表

删除 ETS（Erlang Term Storage）表中的所有对象（所有条目).

ets:delete_all_objects(Table) -> ok

参数

  • Table：ETS 表的名称或标识符。这个名称是你在创建表时指定的。

返回值

  • ok：表示操作成功，即表中的所有对象已被删除。

添加、更新元组

如果插入的数据中已经存在相同的键（key），则插入操作会更新表中原有的值。

  • 如果你使用的是 set 或 ordered_set 类型的表，插入一个已存在的键会覆盖其对应的值。

  • 如果你使用的是 bag 或 duplicate_bag 类型的表，插入相同的键不会替换原来的值，而是允许存在多个相同的键。

-type tab():: atom() | tid().


ets:insert(Tab, ObjectOrObjects) -> true | {error, Reason} when
      Tab :: tab(),
      ObjectOrObjects :: tuple() | [tuple()].

参数

  • Table：需要插入数据的 ETS 表的名称或表的标识符。这个名称是你在创建表时指定的。

  • Object：要插入的对象，可以是一个元组 {Key, Value}，或者在一些特定的情况下多个元组。

返回值：

  • true：表示插入成功。
  • {error, Reason}：表示插入失败，Reason 是错误原因。

复杂查询

用于复杂模式匹配和查询的核心函数

ets:select(Tab, MatchSpec, Limit) -> {[Match], Continuation} | '$end_of_table'

参数

  • Tab：ETS 表的表标识符
  • MatchSpec：匹配规范（match specification）是一个三元组列表，格式为 [{Pattern, Guards, Result}]：
    • Pattern::term()：匹配模式，类似于常规的模式匹配,还有一些特殊的匹配模块符号
      • '_'：匹配任意项但不绑定变量
      • '$N'（如 '$1', '$2'）：绑定匹配项到变量
      • '$_'：引用整个匹配的对象
      • '$$'：引用所有绑定的变量
    • Guards::lists()：保护式（条件表达式），进一步过滤数据
    • Result::lists()：指定返回结果的格式
  • Limit（可选）：限制返回结果数量，用于分页查询

使用辅助函数生成匹配规范

Pattern: 由函数头部的模式匹配转化

Guards: 由函数 when 子句转化

Result : 由函数返回值决定

-include_lib("stdlib/include/ms_transform.hrl").
% 函数式写法生成MatchSpec
find_adults() ->
    ets:fun2ms(fun({person, Name, Age, _}) when Age >= 18 -> Name end).
%% 结果为[{{person,'$1','$2','_'},[{'>','$2',18}],['$1']}]

基本选择

% 创建并填充ETS表
Tab = ets:new(people, [bag]).
ets:insert(Tab, [{person, "Alice", 32, female},
                {person, "Bob", 28, male},
                {person, "Charlie", 35, male}]).

% 选择所有年龄大于30的人
MS = [{{person, '$1', '$2', '_'}, [{'>', '$2', 30}], ['$_']}].
ets:select(Tab, MS).
% 返回: [{person,"Alice",32,female}, {person,"Charlie",35,male}]

带转换的选择

% 选择所有男性，返回姓名和年龄的元组
MS = [{{person, '$1', '$2', male}, [], [{{'$1', '$2'}}]}],
ets:select(Tab, MS).
% 返回: [{"Bob",28}, {"Charlie",35}]

分页查询

% 每次返回1条记录
MS = [{'$1', [], ['$1']}],  % 选择所有记录
{FirstPage, Cont} = ets:select(Tab, MS, 1),
SecondPage = ets:select(Cont).

全量查询

% 获取所有数据（完整记录）
all_data(Tab) ->
    ets:select(Tab, [{'$1', [], ['$1']}]).

% 获取所有数据（特定字段）
all_names(Tab) ->
    ets:select(Tab, [{{'_', '$1', '_', '_'}, [], ['$1']}]).

范围查询

% 年龄在25到35之间
MS = [{{person, '$1', '$2', '_'}, 
      [{'andalso', {'>', '$2', 25}, {'=<', '$2', 35}}], 
      ['$_']}].

多条件查询

% 女性或年龄大于30的男性
MS = [{{person, '$1', '$2', '$3'}, 
      [{'orelse', 
        {'==', '$3', female}, 
        {'andalso', {'==', '$3', male}, {'>', '$2', 30}}
       }], 
      ['$_']}].

结果转换

% 将结果转换为自定义格式
MS = [{{person, '$1', '$2', '$3'}, 
      [], 
      [{[{name, '$1'}, {age, '$2'}, {gender, '$3'}]}]}].

键值查询

查询ETS表中指定键对应的元组

%% 查询指定键对应的元组
ets:lookup(Table, Key) -> [Object].
%% 查询指定键对应的元组中的指定位置的值
ets:lookup_element(Table, Key,Pos) -> term().

参数

  • Table：ETS 表的名称或者标识符。这个名称是你在创建表时指定的。
  • Key：要查找的键，可以是任意 Erlang term（通常是原子、整数或字符串等）。

返回值

  • [Object]：如果找到与 Key 匹配的元素，返回一个包含匹配对象的列表。每个对象通常是一个元组 {Key, Value}，即包含键和值。
  • []：如果没有找到与 Key 匹配的元素，返回一个空列表。

删除元组

从 ETS（Erlang Term Storage）表中删除指定键的函数。如果表中存在与指定键匹配的元素，则删除该元素。

  • 对于 set 和 ordered_set 类型的表，每个键只能对应一个值，因此删除操作会删除匹配该键的元素；
  • 对于 bag 和 duplicate_bag 类型的表，同一键可以有多个值，删除操作会删除匹配该键的所有元素。
ets:delete(Table, Key) -> true | false

参数

  • Table：ETS 表的名称或标识符。这个名称是你在创建表时指定的。
  • Key：要删除的键。可以是任何 Erlang term（通常是原子、整数、字符串等）。

返回值

  • true：表示成功删除了与 Key 匹配的元素。
  • false：表示没有找到与 Key 匹配的元素，删除操作未执行。

查询表信息

获取有关 ETS（Erlang Term Storage） 表的信息。通过该函数，您可以获取表的各种元数据（例如表的类型、访问权限、大小等）。

ets:info(Table) -> term()

参数

  • Table：ETS 表的名称或标识符，这是您在创建表时指定的。

返回值

  • ets:info/1 返回与指定 ETS 表相关的元数据。返回的数据结构通常是一个包含表信息的元组或者键值对。

影响 ETS 表效率的因素

ETS 表在内部是用散列表表示的（除了有序异键表，它是用平衡二叉树表示的）。这就意味着使用异键表会带来少许空间开销，而使用有序异键表会带来时间开销。插入异键表所需的时间是恒定的，而插入有序异键表所需的时间与表内条目数量的对数成比例。 当你在异键表和有序异键表之间进行选择时，应该考虑构建表之后会拿它做什么。如果想要一个排过序的表，就应该使用有序异键表。 使用同键表的代价比使用副本同键表更高，因为每次插入时都需要与所有拥有相同键的元素比较是否相等。如果有大量元组都拥有同一个键，这么做就会很低效。ETS表保存在一个单独的存储区域里，与正常的进程内存无关。可以认为 ETS 表归创建它的进程所有，当这个进程挂了或者调用 ets:delete 时，表就会被删除。ETS 表不会进行垃圾收集，这就意味着即使表里存储了海量的数据也不会产生垃圾收集的开销。 当一个元组被插入 ETS 表时，所有代表这个元组的数据结构都会从进程的栈复制到 ETS 表里。当你对表执行查询操作时，找到的元组会从 ETS 表复制到进程的栈上。

所有的数据结构都是如此，除了大型的二进制数据。这些二进制型会存储在它们自己的堆外存储区域里。这个区域可以被多个进程和 ETS 表共享，各个二进制型则由一个引用计数垃圾收集器管理，它会记录有多少个不同的进程和 ETS 表在使用这个二进制型。如果某个特定二进制型的进程和表使用计数降至零，那么这个二进制型的存储区域就可以被回收。

所有这些听上去可能比较复杂，但最终的结论是：在进程间传递包含大型二进制数据的消息是非常高效的，向 ETS 表插入包含二进制型的元组也非常高效。一个不错的经验法则是尽可能多用二进制型来表示字符串和大块的无类型内存。

DETS

创建表

用于打开一个表，如果表不存在则会创建一个新的表。

dets:open_file(Name, Options) -> {ok, TableId} | {error, Reason}

参数

  • Name：表的名称或路径。这是您想要打开的 DETS 表的文件名，通常是一个字符串，表示文件路径。
  • Options：用于配置表的选项，是一个列表，常见的选项包括：
    • type：表的类型，常见的值有 set（无重复键）和 bag（键可以有多个值）。
    • auto_save：一个布尔值，表示是否启用自动保存功能，默认是 true，即每次修改表都会自动保存。
    • lock：表是否应该被锁定，通常用于确保单个进程在访问表时没有竞争条件。

返回值

  • {ok, TableId}：如果表打开成功，返回 ok 和一个表的标识符 TableId。之后，您可以使用 TableId 进行对表的进一步操作。
  • {error, Reason}：如果打开表失败，返回一个错误元组，Reason 表示错误原因（如表不存在、权限问题等）。

查询表信息

获取与 DETS 表相关的信息

dets:info(TableId) -> term()

参数

  • TableId：通过 dets:open_file/2 打开的表的标识符，表示需要查询信息的 DETS 表。

返回值

dets:info/1 返回一个包含表信息的元组。具体的返回值取决于 DETS 表的状态及其类型。

常见的信息包括：

  • type：表的类型（set 或 bag）。
  • size：表中的元素数量。
  • keypos：键的位置（通常是 1，因为数据是存储为元组）。
  • autobackup：表是否启用了自动备份。
  • auto_save：表是否启用了自动保存（如果启用，数据会在操作后自动写入磁盘）。
  • filename：表的文件名。
  • name：表的名称（如果有的话）。
  • memory：表当前在内存中使用的字节数。

添加、更新元组

将数据插入到 DETS 表

dets:insert(TableId, Object) -> true | {error, Reason}

参数

  • TableId：你通过 dets:open_file/2 打开的表的标识符。TableId 是 dets:open_file/2 返回的值，表示表的句柄。
  • Object：你要插入的数据，可以是任意 Erlang 数据结构，通常是一个元组，如 {Key, Value}。数据的格式取决于你所选择的表的类型（set 或 bag）。

返回值

  • true：表示插入操作成功。
  • {error, Reason}：表示插入操作失败，Reason 表示错误原因。

查找元组

在 DETS（Disk-based Erlang Term Storage） 表中查找特定的键值对

dets:lookup(TableId, Key) -> [Term] | []

参数

  • TableId：你通过 dets:open_file/2 打开的表的标识符。TableId 是 dets:open_file/2 返回的值。
  • Key：你想要查找的键。这个键是存储在表中的项的第一个元素。

返回值

  • [Term]：如果找到了与 Key 匹配的记录，返回一个列表，列表中的每个元素都代表表中与该 Key 匹配的记录。
  • []：如果没有找到与 Key 匹配的记录，返回一个空列表。

删除元组

删除指定键的条目（对象）

dets:delete(TableId, Key) -> ok | {error, Reason}

参数

  • TableId：通过 dets:open_file/2 打开的表的标识符。它表示需要操作的 DETS 表。
  • Key：要删除的条目的键。通常，DETS 表中的数据是以元组 {Key, Value} 的形式存储的，因此，Key 是元组中的第一个元素。

返回值

  • ok：表示删除操作成功。
  • {error, Reason}：表示删除操作失败，Reason 表示失败的原因。

关闭表

关闭一个已打开的 DETS 表

dets:close(TableId) -> ok | {error, Reason}

参数

  • TableId：是你通过调用 dets:open_file/2 打开的表的标识符。TableId 是 dets:open_file/2 返回的值。

返回值

  • ok：表示表已成功关闭。
  • {error, Reason}：如果关闭操作失败，返回一个错误元组，Reason 表示错误原因（例如，badfile，表示传入的 TableId 无效）。

Mnesia

Mnesia 是一种用 Erlang 编写的数据库，它被用于高要求的电信应用程序，同时也是标准 Erlang 分发套装的一部分。将它配置为内存复制后，就能在两个物理隔离的节点上实现快速的容错式数据存储。它还支持事务，并有自己的查询语言。

Mnesia 的速度极快，可以保存任何类型的 Erlang 数据结构。它还是高度可定制的。数据表 既可以保存在内存里（为了速度），也可以保存在磁盘上（为了持久性）。表还可以在不同机器之间进行复制，从而实现容错行为。

创建数据库

在 Erlang 分布式系统中的一个或多个节点上创建 Mnesia 数据库的架构（schema）。

mnesia:create_schema(NodeList).

参数：

  • NodeList：一个包含节点名称的列表，指定在哪些节点上创建 Mnesia 架构。例如，[node1, node2]。

创建

mnesia:create_table/2 是一个用于创建 Mnesia 表的函数。它允许你定义新的表，并指定表的属性，比如主键、存储方式、备份策略等

mnesia:create_table(Name, ArgS).

参数

  • Name：表的名称，是一个原子（atom）。例如，shop 或 user。

  • ArgS：表的属性列表，定义了表的各种属性，如主键、字段等。

    • {attributes, [Field1, Field2, ...]}：定义表中的字段。必须包含主键字段。
    • {type, set | ordered_set | bag | ordered_bag}：定义表的类型。表的类型影响数据如何存储和查询。
      • set：每个主键只能对应一个值。
      • ordered_set：每个主键只能对应一个值，但可以按顺序读取记录。
      • bag：允许多个记录具有相同的主键。
      • ordered_bag：允许多个记录具有相同的主键，并按顺序读取记录。
    • {disc_copies, [Node1, Node2, ...]}：指定在哪些节点上持久化数据。这些节点将保存表的磁盘副本。当我们使用这个选项时，系统还会在执行这个操作的节点上创建一个表的内存副本。
    • {ram_copies, [Node1, Node2, ...]}：指定在哪些节点上存储数据，但数据是内存中的（不持久化）。
    • {index, [Field1, Field2, ...]}：为表定义索引，允许对字段进行加速查询。

QLC

QLC（Quick Lookups） 是 Erlang 中一种查询语言，它允许你对数据进行更复杂和灵活的查询，尤其是在 Mnesia 数据库中。QLC 是一种表达式语言，类似于 SQL，但它是以 Erlang 的列表推导式（List Comprehension）的形式实现的。QLC 使得我们可以执行如过滤、排序、联接等高级查询操作，而且它对多表数据的查询也非常高效。

基本语法
qlc:q([Pattern || Record <- mnesia:table(TableName), Condition]).

  • Pattern：用于匹配记录的模式。

  • Record：从 mnesia:table(TableName) 中提取的每一条记录。

  • Condition：对记录应用的条件，用来过滤结果。

查询表中的所有数据

假设你有一个 shop 表，记录格式为 {id, name, price}，你希望查询所有的记录。

qlc:q([Record || Record <- mnesia:table(shop)]).

这将返回表中所有的记录，因为我们没有在列表推导式中添加任何条件。

查询符合条件的数据

假设你只想查询 price 大于 3 的记录，你可以添加一个条件：

qlc:q([Record || {_, _, Price} <- mnesia:table(shop), Price > 3]).

在这个查询中：

  • {_, _, Price} 是对记录的模式匹配，忽略了 id 和 name，只提取 price 字段。
  • Price > 3 是过滤条件，表示只返回 price 大于 3 的记录。
多表联合查询

假设你有两个表：shop 表（包含商品信息）和 order 表（包含订单信息）。我们希望查询所有购买了 price > 3 商品的客户信息。

假设 order 表的记录格式是 {order_id, customer_id, shop_id, quantity}，你可以通过 shop_id 字段来连接两个表，查询 price > 3 的商品和相关的订单

% SELECT o.customer_id FROM order o JOIN shop s ON o.shop_id = s.id WHERE s.price > 3;

qlc:q([CustomerId || {_, CustomerId, ShopId, _} <- mnesia:table(order),
                    {_, ShopId, Price} <- mnesia:table(shop), Price > 3]).

在这个查询中：

  • 第一个表 order 的记录结构是 {order_id, customer_id, shop_id, quantity}，我们提取了 customer_id 字段。
  • 第二个表 shop 的记录结构是 {id, name, price}，我们提取了 price 字段并筛选出 price > 3 的记录。
分页查询

查询第 n 页的数据，可以通过切片操作在查询结果中模拟分页

Results = qlc:q([Record || Record <- mnesia:table(shop)]),
PageSize = 10,
PageNumber = 2,
Offset = (PageNumber - 1) * PageSize,
PaginatedResults = lists:sublist(Results, Offset + 1, PageSize).
使用排序

查询 shop 表中的所有商品，并按 price 排序

QLC 通过 qlc:sort/2 函数可以对查询结果进行排序

Results = qlc:q([Record || Record <- mnesia:table(shop)]),
qlc:sort(fun({_, _, Price1}, {_, _, Price2}) -> Price1 < Price2 end, Results).
使用聚合操作

计算 shop 表中所有商品的平均 price

Results = qlc:q([Price || {_, _, Price} <- mnesia:table(shop)]),
lists:sum(Results) / length(Results).

  • qlc:q/1 查询了所有商品的 price 字段。

  • lists:sum/1 计算了所有价格的总和。

  • length/1 获取了记录的数量。

  • 最后我们用总和除以数量来计算平均价格。

API

mnesia:dirty_select/2

是一个无事务的查询操作，它会直接返回匹配的所有数据。

mnesia:dirty_select(my_table, [{'_','_','_' }]).

  • my_table 是你要查询的表名。

  • {‘_’, ‘_’, ‘_’} 是一个模式，表示匹配表中每一行的所有字段（_ 是通配符，代表任何值）。

    • 如果你的表有多个字段，可以根据表的结构修改模式。
mnesia:select/2

是一个事务性的查询操作，它会首先启动事务，然后执行查询，并返回结果。

mnesia:transaction(fun() ->
    mnesia:select(my_table, [{'_','_','_'}])
end).
mnesia:read/1

读取指定表中与 Key 匹配的记录。

mnesia:read({Table, Key})

事务

Mnesia 是 Erlang 的分布式数据库系统，支持事务，以确保在并发环境下进行数据操作时的一致性和原子性

Mnesia的事务机制本身依赖函数在失败时抛出异常错误。如果捕捉这些异常错误并试图自行处理它们，会使事务失效。

Mnesia 采用一种悲观锁定（pessimistic locking）的策略。每当 Mnesia 事务管理器访问一个表时，都会根据上下文情况尝试锁定记录甚至整个表。如果它发现这可能导致死锁，就会立即中止事务并撤销之前所做的改动

基本使用

mnesia:transaction(Fun) -> {atomic, Result} | {aborted, Reason}

参数

  • Fun：一个函数，它包含了事务内的数据库操作。这个函数应该是一个闭包，返回一个元组 {atomic, Result} 或 {aborted, Reason}。

返回值

  • Result：事务执行成功后的返回值。
    • 如果事务执行成功并提交，mnesia:transaction/1 会返回 {atomic, Result}；
  • Reason：事务执行失败后的错误原因。
    • 如果事务执行失败并回滚，会返回 {aborted, Reason}。

中止事务

mnesia:abort/1 是用于在事务过程中显式地中止当前事务的操作。当你在事务中调用 mnesia:abort(Reason) 时，Mnesia 会停止当前事务的执行，并将事务的状态回滚到事务开始之前的状态。

mnesia:abort(Reason).

Reason：是一个任意的值，用来描述事务中止的原因。这个值会作为事务回滚时的返回信息，用于通知外部为何事务被中止。

连接数据库--odbc

Erlang 能够连接到传统数据库，例如 SQL Server 和 Oracle。Erlang 有一个 inbuilt odbc library 可用于处理数据库的工具。

建立连接

-module(helloworld). 
-export([start/0]). 

start() ->
   odbc:start(), 
   {ok, Ref} = odbc:connect("DSN = usersqlserver;UID = testuser;PWD = test123", []), 
   io:fwrite("~p",[Ref]).

%输出 <0.33.0>

关于上述程序，需要注意以下几点。

  • odbc 库的启动方法用于指示数据库操作的开始。
  • 连接方法需要 DSN，用户名和密码才能连接。

创建表

-module(helloworld). 
-export([start/0]). 

start() -> 
   odbc:start(), 
   {ok, Ref} = odbc:connect("DSN = usersqlserver; UID = testuser;PWD = test123, []), 
   odbc:sql_query(Ref, "CREATE TABLE EMPLOYEE (
                                            FIRSTNAME char varying(20), 
   											LASTNAME char varying(20),
                                            AGE integer, 
                                            SEX char(1), 
                                            INCOME integer)")

如果现在检查数据库，您将看到将创建一个名为 EMPLOYEE 的表。

插入记录

如果表已成功更新，则记录和语句将返回已更新记录的值和已更新的记录数。

-module(helloworld). 
-export([start/0]). 

start() -> 
   odbc:start(), 
   {ok, Ref} = odbc:connect("DSN = usersqlserver; UID = testuser;PWD = test123", []), 
   io:fwrite("~p",[odbc:sql_query(Ref, 
   "INSERT INTO EMPLOYEE VALUES('Mac', 'Mohan', 20, 'M', 2000)")]).
% 输出  {updated,1}
