请求对象

约 2295 字大约 8 分钟

2025-03-28

Req 对象是一个用于获取请求信息、读取其主体或发送响应的变量。

它并不是真正意义上的面向对象中的对象。它是一个简单的映射，可以直接访问或在调用 cowboy_req 模块中的函数时使用。

Req 对象是几个不同章节的主题。在本章中，我们将学习 Req 对象并查看如何获取请求信息。

直接访问

Req map 包含了一些字段，这些字段是文档化的，并可以直接访问。它们是直接映射到 HTTP 的字段：请求 method；使用的 HTTPversion；有效的 URI 组件 scheme、host、port、path 和 qs；请求 headers；连接 peer 地址和端口；以及当适用时的 TLS 证书 cert。

请注意，version 字段可以用来确定连接是否使用 HTTP/2。

要访问一个字段，您只需在函数头部进行匹配。以下示例在 method 为 GET 时发送一个简单的"Hello world!"响应，否则发送一个 405 错误。

init(Req0=#{method := <<"GET">>}, State) ->
    Req = cowboy_req:reply(200, #{
        <<"content-type">> => <<"text/plain">>
    }, <<"Hello world!">>, Req0),
    {ok, Req, State};
init(Req0, State) ->
    Req = cowboy_req:reply(405, #{
        <<"allow">> => <<"GET">>
    }, Req0),
    {ok, Req, State}.

其他任何字段都是内部字段，不应访问。它们可能会在未来版本中，包括维护版本中，未经通知就发生变化。

修改 Req 对象是可以的，但在修改现有字段时必须格外小心。可以添加任意数量的新字段，但请确保命名空间字段名称，以避免与未来的 Cowboy 更新或第三方项目发生冲突。

cowboy_req 接口介绍

cowboy_req 模块中的函数提供了对请求信息的访问，同时也提供了处理 HTTP 请求时常见的各种操作。

所有以动词开头的函数表示一个动作。其他函数则只是返回相应的值（有时需要构建这个值，但操作的成本等同于检索一个值）。

一些 cowboy_req 函数会返回一个更新的 Req 对象。这些函数包括 read、reply、set 和 delete。虽然忽略返回的 Req 对象在某些情况下不会导致不正确的行为，但强烈建议始终保留并使用最后返回的 Req 对象。cowboy_req 的手册详细描述了这些函数以及对 Req 对象所做的修改。

一些对 cowboy_req 的调用具有副作用。read 和 reply 函数就是这种情况。Cowboy 在函数被调用时会立即读取请求体或立即回复。

如果发生错误，所有函数都会崩溃。通常不需要捕获这些错误，Cowboy 会根据错误发生的位置发送适当的 4xx 或 5xx 响应。

请求方法

请求方法可以直接获取：

#{method := Method} = Req.

或者使用函数：

Method = cowboy_req:method(Req).

方法是一个区分大小写的二进制字符串。标准方法包括 GET、HEAD、OPTIONS、PATCH、POST、PUT 或 DELETE。

HTTP 版本

HTTP 版本是信息性的。它并不表示客户端实现该协议良好或完全。

通常情况下，不需要根据 HTTP 版本改变行为：Cowboy 已经为你做了。

但在某些情况下，这可能是有用的。例如，可以将 HTTP/1.1 客户端重定向到使用 Websocket，而 HTTP/2 客户端继续使用 HTTP/2。

可以直接获取 HTTP 版本：

#{version := Version} = Req.

或者使用一个函数：

Version = cowboy_req:version(Req).

Cowboy 定义了 'HTTP/1.0'，'HTTP/1.1' 和 'HTTP/2' 版本。自定义协议可以将其自己的值定义为原子值。

有效的请求 URI

有效的请求 URI 的 scheme，host，port，path 和 query string 组件都可以直接获取：

#{
    scheme := Scheme,
    host := Host,
    port := Port,
    path := Path,
    qs := Qs
} = Req.

或使用相关的函数：

Scheme = cowboy_req:scheme(Req),
Host = cowboy_req:host(Req),
Port = cowboy_req:port(Req),
Path = cowboy_req:path(Req).
Qs = cowboy_req:qs(Req).

方案和主机是不区分大小写的二进制字符串。端口是一个表示端口号的整数。路径和查询字符串是区分大小写的二进制字符串。

Cowboy 只定义了 <<"http">> 和 <<"https">> 方案。这些方案被选择为确保在安全的 HTTP/1.1 或 HTTP/2 连接上请求的方案将只有 <<"https">>。

有效的请求 URI 可以通过 cowboy_req:uri/1,2 函数重新构建。默认情况下，返回的是绝对 URI：

%% scheme://host[:port]/path[?qs]
URI = cowboy_req:uri(Req).

选项可以禁用或替换一些或全部组件。可以生成各种 URI 或 URI 格式，包括原始形式：

%% /path[?qs]
URI = cowboy_req:uri(Req, #{host => undefined}).

协议相对形式：

%% //host[:port]/path[?qs]
URI = cowboy_req:uri(Req, #{scheme => undefined}).

不含查询字符串的绝对 URI：

URI = cowboy_req:uri(Req, #{qs => undefined}).

不同的主机:

URI = cowboy_req:uri(Req, #{host => <<"example.org">>}).

以及其他组合。

绑定

绑定是您在定义应用程序路由时选择提取的主机和路径组件。它们仅在路由之后可用。

Cowboy 提供了获取一个或所有绑定的功能。

要获取单个值：

Value = cowboy_req:binding(userid, Req).

当尝试获取未绑定的值时，将会返回 undefined。可以提供一个不同的默认值：

Value = cowboy_req:binding(userid, Req, 42).

要获取所有绑定的内容：

Bindings = cowboy_req:bindings(Req).

它们以映射形式返回，键为原子。

牛仔路由器还允许您使用 ... 限定符一次捕获主机或路径中的多个段。

要检索从主机名捕获的段：

HostInfo = cowboy_req:host_info(Req).

以及路径段：

PathInfo = cowboy_req:path_info(Req).

如果未在路由中使用 ...，Cowboy 将返回 undefined。

查询参数

Cowboy 提供了两个函数来访问查询参数。您可以使用第一个函数来获取所有参数的完整列表。

QsVals = cowboy_req:parse_qs(Req),
{_, Lang} = lists:keyfind(<<"lang">>, 1, QsVals).

Cowboy 只会解析查询字符串，不会进行任何转换。因此，返回的列表可能会包含重复项，或者参数名称没有关联的值。返回的列表的顺序是未定义的。

当查询字符串为 key=1&key=2 时，返回的列表将包含两个名为 key 的参数。

当尝试使用 PHP 风格的后缀 [] 时也是如此。当查询字符串为 key[]=1&key[]=2 时，返回的列表将包含两个名为 key[] 的参数。Cowboy 不需要 [] 后缀即可正确处理重复的键名。

当查询字符串仅为 key 时，Cowboy 将返回列表 [{<<"key">>, true}]，使用 true 表示参数 key 已定义但没有值。

Cowboy提供的第二个函数允许你仅匹配出感兴趣的参数，并且可以使用 约束 完成任何后续处理。此函数返回一个映射。

#{id := ID, lang := Lang} = cowboy_req:match_qs([id, lang], Req).

约束可以自动应用。以下代码片段将在 id 参数不是整数或 lang 参数为空时崩溃。同时，id 的值将被转换为整数类型：

QsMap = cowboy_req:match_qs([{id, int}, {lang, nonempty}], Req).

默认值也可以提供。如果找不到 lang 键，将使用默认值。如果找到了键但其值为空，则不会使用默认值。

#{lang := Lang} = cowboy_req:match_qs([{lang, [], <<"en-US">>}], Req).

如果没有提供默认值且值缺失，查询字符串将被视为无效，导致进程崩溃。

当查询字符串为 key=1&key=2 时，key 的值将是一个列表 [<<"1">>, <<"2">>]。参数名称不需要包含 PHP 风格的后缀。可以通过约束确保只给定了一个值。约束不会自动检查列表内部，如果需要这样做，则必须编写自定义约束。

头部

头部值可以以二进制字符串的形式获取，也可以解析为更具意义的表示形式。

要获取原始值：

HeaderVal = cowboy_req:header(<<"content-type">>, Req).

Cowboy 要求所有头部名称都以小写的二进制字符串形式提供。无论是请求还是响应，这都适用于所有底层协议。

如果请求中缺少该头部，将返回 undefined。可以提供不同的默认值：

HeaderVal = cowboy_req:header(<<"content-type">>, Req, <<"text/plain">>).

所有头部可以一次性获取，可以直接获取：

#{headers := AllHeaders} = Req.

或者使用一个函数：

AllHeaders = cowboy_req:headers(Req).

牛仔提供了解析单个头的等效函数。没有一次解析所有头的功能。

要解析特定的头：

ParsedVal = cowboy_req:parse_header(<<"content-type">>, Req).

如果不知道如何解析给定的头，或者值无效，将会抛出异常。已知头和默认值的列表可以在手册中找到。

如果头缺失，将返回 undefined。你可以更改默认值。请注意，应该是解析后的值直接：

ParsedVal = cowboy_req:parse_header(<<"content-type">>, Req,
    {<<"text">>, <<"plain">>, []}).

Peer

连接的对端地址和端口号可以直接获取或使用函数获取。

直接获取对端：

#{peer := {IP, Port}} = Req.

使用函数获取：

{IP, Port} = cowboy_req:peer(Req).

请注意，peer 对应到连接到服务器的远程端点，可能是客户端本身，也可能是代理或网关。
