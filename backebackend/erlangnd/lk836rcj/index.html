包含的应用程序

约 917 字大约 3 分钟

2025-03-22

一个应用程序可以包含其他应用程序。一个包含的应用程序有自己的应用程序目录和.app文件，但它作为另一个应用程序的监督树的一部分启动。

一个应用程序只能被一个其他应用程序包含。

被包含的应用程序可以包含其他应用程序。

没有被任何其他应用程序包含的应用程序称为一个 主要应用.

应用程序控制器在加载主应用程序时会自动加载任何包含的应用程序，但不会启动它们。相反，包含应用程序中的监督者必须启动包含应用程序的顶级监督者。

这意味着运行时，一个包含的应用实际上是主应用的一部分，而包含应用中的进程认为自己属于主应用。

指定包含的应用程序

包含的应用程序由included_applications键在.app文件中定义：

{application, prim_app,
 [{description, "Tree application"},
  {vsn, "1"},
  {modules, [prim_app_cb, prim_app_sup, prim_app_server]},
  {registered, [prim_app_server]},
  {included_applications, [incl_app]},
  {applications, [kernel, stdlib, sasl]},
  {mod, {prim_app_cb,[]}},
  {env, [{file, "/usr/local/log"}]}
 ]}.

启动时同步进程

包含的应用程序的监督树作为包含应用程序的监督树的一部分启动。如果需要在包含应用程序和包含应用程序之间进行进程同步，可以通过使用启动阶段来实现。

启动阶段通过start_phases键在.app文件中定义为元组列表{Phase,PhaseArgs}，其中Phase是一个原子，PhaseArgs是一个术语。

包含应用程序的 mod 键的值必须设置为 {application_starter,[Module,StartArgs]} ，其中 Module 通常是指应用程序回调模块。StartArgs 是作为 Module:start/2 回调函数参数提供的一个术语：

{application, prim_app,
 [{description, "Tree application"},
  {vsn, "1"},
  {modules, [prim_app_cb, prim_app_sup, prim_app_server]},
  {registered, [prim_app_server]},
  {included_applications, [incl_app]},
  {start_phases, [{init,[]}, {go,[]}]},
  {applications, [kernel, stdlib, sasl]},
  {mod, {application_starter,[prim_app_cb,[]]}},
  {env, [{file, "/usr/local/log"}]}
 ]}.

{application, incl_app,
 [{description, "Included application"},
  {vsn, "1"},
  {modules, [incl_app_cb, incl_app_sup, incl_app_server]},
  {registered, []},
  {start_phases, [{go,[]}]},
  {applications, [kernel, stdlib, sasl]},
  {mod, {incl_app_cb,[]}}
 ]}.

在启动具有包含应用程序的主应用程序时，主应用程序将像通常那样启动，即：

  • 应用控制器为应用程序创建一个应用程序主控
  • 应用程序主控调用 Module:start(normal, StartArgs) 来启动顶级监督者。

然后，对于主应用程序和顶级向下中的每个包含应用程序， 从左到右的顺序，应用主调用 Module:start_phase(Phase, Type, PhaseArgs) 对主应用定义的每个阶段，按照这个顺序依次调用。如果未为包含的应用定义某个阶段，则不会为该阶段和应用调用此函数。

以下要求适用于包含应用的.app文件：

  • 必须包含 {mod, {Module,StartArgs}} 选项。此选项用于查找应用程序的回调模块 Module。StartArgs 被忽略，因为仅对主应用程序调用 Module:start/2。
  • 如果包含的应用程序本身也包含其他包含的应用程序，则 必须包含 {mod, {application_starter, [Module,StartArgs]}} 选项。
  • {start_phases, [{Phase,PhaseArgs}]} 选项必须包含，并且指定的阶段集合必须是为主应用程序指定的阶段集合的子集。

当以上定义启动prim_app时，在application:start(prim_app)返回值之前，应用程序控制器会调用以下回调函数：

application:start(prim_app)
 => prim_app_cb:start(normal, [])
 => prim_app_cb:start_phase(init, normal, [])
 => prim_app_cb:start_phase(go, normal, [])
 => incl_app_cb:start_phase(go, normal, [])
ok
