函数式编程

约 1165 字大约 4 分钟

erlang

2025-09-24

概念

提示

"函数式编程"是一种"编程范式"（programming paradigm），也就是如何编写程序的方法论。

它属于"结构化编程"的一种，主要思想是把运算过程尽量写成一系列嵌套的函数调用。 ---《百度百科》

特点

函数是"第一等公民"

所谓"第一等公民"（first class），指的是函数与其他数据类型一样，处于平等地位，可以赋值给其他变量，也可以作为参数，传入另一个函数，或者作为别的函数的返回值。

% 函数可以赋值给变量
Double = fun(X) -> X * 2 end,
Triple = fun(X) -> X * 3 end.

% 函数作为参数传递
apply_twice(F, X) -> F(F(X)).

% 调用
Result = apply_twice(Double, 5). % 返回 20 (5*2*2)

% 函数作为返回值
make_multiplier(N) -> fun(X) -> X * N end.

Times4 = make_multiplier(4),
Times4(5). % 返回 20

只用"表达式"，不用"语句"

"表达式"（expression）是一个单纯的运算过程，总是有返回值；"语句"（statement）是执行某种操作，没有返回值。函数式编程要求，只使用表达式，不使用语句。也就是说，每一步都是单纯的运算，而且都有返回值。

% 错误的方式（语句式思维）
set_value(X) ->
    Y = X + 1,     % 这看起来像语句，但在Erlang中也是表达式
    Y.             % 最后一行必须是表达式

% 正确的函数式方式（一切都是表达式）
calculate(X) -> X + 1.

% if, case 都是表达式，都有返回值
get_grade(Score) ->
    if 
        Score >= 90 -> excellent;
        Score >= 60 -> pass;
        true -> fail
    end.  % 整个if返回一个原子值

Grade = get_grade(85). % Grade = pass

没有"副作用"

所谓"副作用"（side effect），指的是函数内部与外部互动（最典型的情况，就是修改全局变量的值），产生运算以外的其他结果。

函数式编程强调没有"副作用"，意味着函数要保持独立，所有功能就是返回一个新的值，没有其他行为，尤其是不得修改外部变量的值。

% 有副作用的做法（非函数式）
-module(bad_example).
-export([increment/0]).

-initialize_counter() -> put(counter, 0).  % 修改进程字典（副作用）

increment() -> 
    put(counter, get(counter) + 1).  % 修改外部状态

% 函数式做法（无副作用）
-module(good_example).
-export([increment/1]).

increment(Counter) -> Counter + 1.  % 不修改输入，返回新值

% 使用
NewCounter = increment(OldCounter).  % 原始Counter不会被修改

不修改状态

状态一般是通过变量来保存，不修改状态意味着不能修改输入值，而是每次返回输出值，通过输出保存状态。

% 非函数式（修改状态）
-module(mutable_state).
-export([add_item/2]).

add_item(Item, List) ->
    List ++ [Item].  % 看起来没修改？实际上在Erlang中数据不可变

% 实际上在Erlang中，所有数据都是不可变的
List1 = [1, 2, 3],
List2 = List1 ++ [4],  % List1 保持不变，List2 是新列表

% 状态通过函数参数传递和返回
process_list([], Acc) -> lists:reverse(Acc);
process_list([H|T], Acc) when H > 0 ->
    process_list(T, [H * 2 | Acc]);  % 不修改原列表，构建新列表
process_list([_|T], Acc) ->
    process_list(T, Acc).

引用透明

引用透明（Referential transparency），指的是函数的运行不依赖于外部变量或"状态"，只依赖于输入的参数，任何时候只要参数相同，引用函数所得到的返回值总是相同的。

% 引用透明函数
add(X, Y) -> X + Y.

% 任何时候调用 add(2, 3) 都返回 5
% 可以被替换为结果值而不改变程序行为

% 非引用透明函数
get_timestamp() -> erlang:system_time(millisecond).
% 每次调用返回不同值，依赖于外部状态

% 测试引用透明性
% add(2, 3) 在任何地方都可以直接替换为 5
Result1 = add(2, 3) + add(2, 3),     % 10
Result2 = 5 + 5,                      % 10 - 结果相同
% 两者完全等价

惰性计算

在惰性计算中，表达式不是在绑定到变量时立即计算，而是在求值程序需要产生表达式的值时进行计算。延迟的计算使您可以编写可能潜在地生成无穷输出的函数。因为不会计算多于程序的其余部分所需要的值，所以不需要担心由无穷计算所导致的 out-of-memory 错误。


%% 如果Award为整数，这个语句就会报错{'EXIT',badarg}，因为函数调用前会先计算所有参数的值
game_lib:three(is_integer(Award), [Award | Acc], Award++Acc).
%% 这个语句不会报错，函数参数不会立即执行，而是在使用到该函数参数时才会进行执行
game_lib:three(is_integer(Award), [Award | Acc], fun() -> Award ++ Acc end).
