B-树：概念

约 1289 字大约 4 分钟

2025-12-09

3.1 B-树和 BST 的直觉

我们的第一个直觉来自平衡二叉树（BST）。二叉树是排序数据的流行数据结构。在插入或删除键后保持树的良好形状就是"平衡"的含义。如前一章所述，应该使用 n 叉树而不是二叉树，以利用"页面"（IO 的最小单位）。

B-树可以从 BST 推广而来。B-树的每个节点包含多个键和多个指向其子节点的链接。在节点中查找键时，使用所有键来决定下一个子节点。

        [1, 4, 9]
          / | \
         /  |  \
       v    v    v
[1, 2, 3] [4, 6] [9, 11, 12]

B-树的平衡与 BST 不同，流行的 BST 如红黑树或 AVL 树通过旋转在子树的高度上保持平衡。而所有 B-树叶节点的高度相同，B-树通过节点的大小保持平衡：

  • 如果一个节点太大，无法容纳在一个页面上，它会被拆分为两个节点。这将增加父节点的大小，并可能在根节点被拆分时增加树的高度。
  • 如果一个节点太小，尝试将其与兄弟节点合并。

如果你熟悉红黑树，你可能也知道 2-3 树，它可以很容易地推广为 B-树。

3.2 B-树和嵌套数组

即使你不熟悉 2-3 树，你仍然可以使用嵌套数组获得一些直觉。

让我们从一个排序数组开始。查询可以通过二分查找完成。但是，更新数组是 O(n)，这是我们需要解决的问题。更新一个大数组很糟糕，所以我们把它分成更小的数组。假设我们将数组分成 sqrt(n) 部分，每部分平均包含 sqrt(n) 个键。

[[1,2,3], [4,6], [9,11,12]]

要查询一个键，我们必须首先确定哪个部分包含该键，在 sqrt(n) 个部分上进行二分查找是 O(log(n))。之后，在该部分上对键进行二分查找又是 O(log(n))——不比以前差。更新改进为 O(sqrt(n))。

这是一个 2 级排序嵌套数组，如果我们添加更多层级呢？这是 B-树的另一种直觉。

3.3 B-树操作

查询 B-树与查询 BST 相同。

更新 B-树更复杂。从现在开始，我们将使用一种称为"B+ 树"的 B-树变体，B+ 树只在叶节点中存储值，内部节点只包含键。

键插入从叶子开始。叶子只是一个排序的键列表。将键插入叶子是微不足道的。但是，插入可能导致节点大小超过页面大小。在这种情况下，我们需要将叶子节点拆分为 2 个节点，每个节点包含一半的键，以便两个叶子节点都能容纳在一个页面中。

内部节点由以下组成：

  1. 指向其子节点的指针列表。
  2. 与指针列表配对的键列表。每个键是对应子节点的第一个键。

将叶子节点拆分为 2 个节点后，父节点用新的指针和键替换旧的指针和键。节点的大小增加，可能触发进一步的拆分。

parent               parent
 / | \       =>     / | | \
L1 L2 L6           L1 L3 L4 L6

根节点被拆分后，会添加一个新的根节点。这就是 B-树的生长方式。

new_root
   |
   root
  / | \       =>     / | | \
 L1 L2 L6           L1 L3 L4 L6

键删除是插入的反向操作。节点永远不会为空，因为一个小节点将被合并到其左兄弟或右兄弟中。

当非叶根节点减少到只有一个键时，根节点可以被其唯一的子节点替换。这就是 B-树的收缩方式。

3.4 不可变数据结构

不可变意味着从不原地更新数据。一些类似的术语是"仅追加"、“写时复制"和"持久数据结构”（"持久"这个词与我们之前讨论的"持久化"无关）。

例如，当将键插入叶子节点时，不要原地修改节点，而是创建一个新节点，其中包含来自待更新节点的所有键和新键。现在父节点也必须更新以指向新节点。

同样，父节点被复制并带有新指针。直到我们到达根节点，整个路径都被复制了。这有效地创建了一个与旧版本共存的新版本树。我们之前提到的 LSM-树也被认为是不可变的。

不可变数据结构有几个优点：

  1. 避免数据损坏。不可变数据结构不修改现有数据，它们只是添加新数据，因此即使更新被中断，旧版本的数据也保持不变。
  2. 易于并发。读取器可以与写入器并发操作，因为读取器可以在不受影响的旧版本上工作。

持久化和并发将在后面的章节中介绍。现在，我们将首先编写一个不可变的 B+ 树。
