持久化到磁盘

约 2347 字大约 8 分钟

2025-12-09

上一章的 B-树数据结构可以轻松转储到磁盘。让我们在其之上构建一个简单的键值存储。由于我们的 B-树实现是不可变的，我们将以仅追加的方式分配磁盘空间，磁盘空间的重用推迟到下一章。

6.1 持久化数据的方法

如前面章节所述，将数据持久化到磁盘不仅仅是把数据转储到文件中。有几个考虑因素：

  1. 崩溃恢复：这包括数据库进程崩溃、操作系统崩溃和断电。数据库在重启后必须处于可用状态。
  2. 持久性：在数据库成功响应后，所涉及的数据保证持久化，即使在崩溃后。换句话说，持久化发生在响应客户端之前。

有很多资料使用 ACID 术语（原子性、一致性、隔离性、持久性）描述数据库，但这些概念不是正交的且难以解释，所以让我们专注于我们的实际例子。

  1. 我们 B-树的不可变方面：更新 B-树不会触及先前版本的 B-树，这使得崩溃恢复变得容易——如果更新出错，我们可以简单地恢复到先前版本。
  2. 持久性通过 Linux 系统调用 fsync 实现。通过 write 或 mmap 的正常文件 IO 首先进入页面缓存，系统稍后必须将页面缓存刷新到磁盘。fsync 系统调用会阻塞，直到所有脏页被刷新。

如果更新出错，我们如何恢复到先前版本？我们可以将更新分为两个阶段：

  1. 更新创建新节点；将它们写入磁盘。
  2. 每次更新创建一个新的根节点，我们需要将指向根节点的指针存储在某个地方。

第一阶段可能涉及将多个页面写入磁盘，这通常不是原子的。但第二阶段只涉及单个指针，可以通过原子的单页写入完成。这使得整个操作具有原子性——如果数据库崩溃，更新根本不会发生。

第一阶段必须在第二阶段之前持久化，否则，根指针可能在崩溃后指向树的损坏（部分持久化）版本。两个阶段之间应该有一个 fsync（作为屏障）。

第二阶段也应在响应客户端之前进行 fsync。

6.2 基于 mmap 的 IO

磁盘文件的内容可以使用 mmap 系统调用从虚拟地址映射。从这个地址读取会启动透明的磁盘 IO，与通过 read 系统调用读取文件相同，但不需要用户空间缓冲区和系统调用的开销。映射地址是页面缓存的代理，通过它修改数据与 write 系统调用相同。

mmap 很方便，我们将为我们的键值存储使用它。然而，使用 mmap 并不是必需的。

// 创建覆盖整个文件的初始 mmap。
func mmapInit(fp *os.File) (int, []byte, error) {
    fi, err := fp.Stat()
    if err != nil {
        return 0, nil, fmt.Errorf("stat: %w", err)
    }

    if fi.Size()%BTREE_PAGE_SIZE != 0 {
        return 0, nil, errors.New("File size is not a multiple of page size.")
    }

    mmapSize := 64 << 20
    assert(mmapSize%BTREE_PAGE_SIZE == 0)
    for mmapSize < int(fi.Size()) {
        mmapSize *= 2
    }
    // mmapSize 可以大于文件大小

    chunk, err := syscall.Mmap(
        int(fp.Fd()), 0, mmapSize,
        syscall.PROT_READ|syscall.PROT_WRITE, syscall.MAP_SHARED,
    )
    if err != nil {
        return 0, nil, fmt.Errorf("mmap: %w", err)
    }

    return int(fi.Size()), chunk, nil
}

上述函数创建至少文件大小的初始映射。映射的大小可以大于文件大小，文件末尾之后的范围不可访问（SIGBUS），但文件稍后可以扩展。

随着文件增长，我们可能必须扩展映射范围。扩展 mmap 范围的系统调用是 mremap。不幸的是，通过重新映射扩展范围时，我们可能无法保持起始地址。我们扩展映射的方法是使用多个映射——为溢出的文件范围创建新的映射。

type KV struct {
    Path string
    // 内部
    fp   *os.File
    tree BTree
    mmap struct {
        file   int      // 文件大小，可以大于数据库大小
        total  int      // mmap 大小，可以大于文件大小
        chunks [][]byte // 多个 mmap，可以是不连续的
    }
    page struct {
        flushed uint64    // 数据库大小（页面数）
        temp    [][]byte  // 新分配的页面
    }
}

// 通过添加新映射来扩展 mmap。
func extendMmap(db *KV, npages int) error {
    if db.mmap.total >= npages*BTREE_PAGE_SIZE {
        return nil
    }

    // 地址空间加倍
    chunk, err := syscall.Mmap(
        int(db.fp.Fd()), int64(db.mmap.total), db.mmap.total,
        syscall.PROT_READ|syscall.PROT_WRITE, syscall.MAP_SHARED,
    )
    if err != nil {
        return fmt.Errorf("mmap: %w", err)
    }

    db.mmap.total += db.mmap.total
    db.mmap.chunks = append(db.mmap.chunks, chunk)
    return nil
}

新映射的大小呈指数增长，这样我们就不必频繁调用 mmap。

以下是我们如何从映射地址访问页面。

// BTree 的回调，解引用指针。
func (db *KV) pageGet(ptr uint64) BNode {
    start := uint64(0)
    for _, chunk := range db.mmap.chunks {
        end := start + uint64(len(chunk))/BTREE_PAGE_SIZE
        if ptr < end {
            offset := BTREE_PAGE_SIZE * (ptr - start)
            return BNode{chunk[offset : offset+BTREE_PAGE_SIZE]}
        }
        start = end
    }
    panic("bad ptr")
}

6.3 主页面

文件的第一个页面用于存储指向根的指针，我们称之为"主页面"。页面总数用于分配新节点，因此也存储在那里。

| 主页面 | 页面... | 树根 | 页面... |
| btree_root | page_used | ^ ^
|            |           | |
+------------+----------------------+ |
|                                    |
+--------------------------------复制

下面的函数在初始化数据库时读取主页面：

const DB_SIG = "BuildYourOwnDB06"

// 主页面格式。
// 它包含指向根的指针和其他重要位。
// | 签名 | btree_root | page_used |
// | 16B  | 8B         | 8B        |
func masterLoad(db *KV) error {
    if db.mmap.file == 0 {
        // 空文件，主页面将在第一次写入时创建。
        db.page.flushed = 1 // 为主页面保留
        return nil
    }

    data := db.mmap.chunks[0]
    root := binary.LittleEndian.Uint64(data[16:])
    used := binary.LittleEndian.Uint64(data[24:])

    // 验证页面
    if !bytes.Equal([]byte(DB_SIG), data[:16]) {
        return errors.New("Bad signature.")
    }

    bad := !(1 <= used && used <= uint64(db.mmap.file/BTREE_PAGE_SIZE))
    bad = bad || !(0 <= root && root < used)
    if bad {
        return errors.New("Bad master page.")
    }

    db.tree.root = root
    db.page.flushed = used
    return nil
}

下面是更新主页面的函数。与读取代码不同，它不使用映射地址进行写入。这是因为通过 mmap 修改页面不是原子的。内核可能在写入中途刷新页面并损坏磁盘文件，而不跨越页面边界的小写入保证是原子的。

// 更新主页面。必须是原子的。
func masterStore(db *KV) error {
    var data [32]byte
    copy(data[:16], []byte(DB_SIG))
    binary.LittleEndian.PutUint64(data[16:], db.tree.root)
    binary.LittleEndian.PutUint64(data[24:], db.page.flushed)

    // 注意：通过 mmap 更新页面不是原子的。
    // 使用 `pwrite()` 系统调用代替。
    _, err := db.fp.WriteAt(data[:], 0)
    if err != nil {
        return fmt.Errorf("write master page: %w", err)
    }
    return nil
}

6.4 分配磁盘页面

在下一章添加空闲列表之前，我们将简单地将新页面追加到数据库的末尾。

新页面暂时保存在内存中，直到稍后复制到文件中（可能在扩展文件之后）。

// BTree 的回调，分配新页面。

在写入待处理页面之前，我们可能需要先扩展文件。相应的系统调用是 fallocate。

// 将文件扩展到至少 `npages` 页。
func extendFile(db *KV, npages int) error {
    filePages := db.mmap.file / BTREE_PAGE_SIZE
    if filePages >= npages {
        return nil
    }

    for filePages < npages {
        // 文件大小呈指数增长，
        // 这样我们就不必为每次更新扩展文件。
        inc := filePages / 8
        if inc < 1 {
            inc = 1
        }
        filePages += inc
    }

    fileSize := filePages * BTREE_PAGE_SIZE
    err := syscall.Fallocate(int(db.fp.Fd()), 0, 0, int64(fileSize))
    if err != nil {
        return fmt.Errorf("fallocate: %w", err)
    }

    db.mmap.file = fileSize
    return nil
}

6.5 初始化数据库

将我们已完成的部分组合起来。

func (db *KV) Open() error {
    // 打开或创建数据库文件
    fp, err := os.OpenFile(db.Path, os.O_RDWR|os.O_CREATE, 0644)
    if err != nil {
        return fmt.Errorf("OpenFile: %w", err)
    }
    db.fp = fp

    // 创建初始 mmap
    sz, chunk, err := mmapInit(db.fp)
    if err != nil {
        goto fail
    }
    db.mmap.file = sz
    db.mmap.total = len(chunk)
    db.mmap.chunks = [][]byte{chunk}

    // btree 回调
    db.tree.get = db.pageGet
    db.tree.new = db.pageNew
    db.tree.del = db.pageDel

    // 读取主页面
    err = masterLoad(db)
    if err != nil {
        goto fail
    }

    // 完成
    return nil

fail:
    db.Close()
    return fmt.Errorf("KV.Open: %w", err)
}

// 清理
func (db *KV) Close() {
    for _, chunk := range db.mmap.chunks {
        err := syscall.Munmap(chunk)
        assert(err == nil)
    }
    _ = db.fp.Close()
}

6.6 更新操作

与查询不同，更新操作必须在返回之前持久化数据。

// 读取数据库
func (db *KV) Get(key []byte) ([]byte, bool) {
    return db.tree.Get(key)
}

// 更新数据库
func (db *KV) Set(key []byte, val []byte) error {
    db.tree.Insert(key, val)
    return flushPages(db)
}

func (db *KV) Del(key []byte) (bool, error) {
    deleted := db.tree.Delete(key)
    return deleted, flushPages(db)
}

flushPages 是持久化新页面的函数。

// 更新后持久化新分配的页面
func flushPages(db *KV) error {
    if err := writePages(db); err != nil {
        return err
    }
    return syncPages(db)
}

如前所述，它分为两个阶段。

func writePages(db *KV) error {
    // 如果需要，扩展文件 & mmap
    npages := int(db.page.flushed) + len(db.page.temp)
    if err := extendFile(db, npages); err != nil {
        return err
    }
    if err := extendMmap(db, npages); err != nil {
        return err
    }

    // 将数据复制到文件
    for i, page := range db.page.temp {
        ptr := db.page.flushed + uint64(i)
        copy(db.pageGet(ptr).data, page)
    }
    return nil
}

fsync 在它们之间和之后。

func syncPages(db *KV) error {
    // 将数据刷新到磁盘。必须在更新主页面之前完成。
    if err := db.fp.Sync(); err != nil {
        return fmt.Errorf("fsync: %w", err)
    }

    db.page.flushed += uint64(len(db.page.temp))
    db.page.temp = db.page.temp[:0]

    // 更新并刷新主页面
    if err := masterStore(db); err != nil {
        return err
    }
    if err := db.fp.Sync(); err != nil {
        return fmt.Errorf("fsync: %w", err)
    }
    return nil
}
