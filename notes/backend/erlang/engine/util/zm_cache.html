zm_cache---数据缓冲模块

约 1273 字大约 4 分钟

2026-01-22

模块概述

zm_cache 是一个数据缓冲模块，基于ETS表实现的内存缓存系统。该模块提供了缓存数据的创建、存储、检索、清理和销毁等功能，支持基于时间和内存大小的自动清理策略。

核心特性：

  • 基于ETS的内存缓存
  • 支持超时清理（TTL）
  • 支持内存大小限制
  • 自动清理策略
  • 读取时间更新选项
  • 批量操作支持

缓存配置说明

缓存参数

  1. Sizeout（内存限制）
    • 单位：千字节（KB）
    • Sizeout ≤ 0：不缓冲数据
    • Sizeout = infinity：无内存大小限制
    • 当内存超过Sizeout时，按时间段将老数据清除，每次将大小降为当前内存一半左右
  2. Timeout（超时时间）
    • 单位：毫秒（ms）
    • 必须为正整数
    • Timeout ≥ 4294967295（约49.8天）：表示永不超期
    • 每次整理先将超时数据清除，然后清除老数据
  3. ReadTimeUpdate（读取时间更新）
    • 布尔值，默认为true
    • 控制读取操作是否更新数据的访问时间

缓存结构

Cache = {Ets, Sizeout, Timeout, ReadTimeUpdate}
复制
  • Ets：ETS表标识符
  • Sizeout：内存限制
  • Timeout：超时时间
  • ReadTimeUpdate：读取时间更新标志

缓存管理函数

new/2

new(Sizeout::integer(), Timeout::integer()) -> Cache::cache()

创建新缓存。

参数：

  • Sizeout：内存限制（KB）
  • Timeout：超时时间（ms）

返回值： 缓存句柄

new/3

new(Sizeout::integer(), Timeout::integer(), ReadTimeUpdate::boolean()) -> Cache::cache()

创建新缓存（可指定读取时间更新选项）。

参数：

  • Sizeout：内存限制（KB）
  • Timeout：超时时间（ms）
  • ReadTimeUpdate：是否在读取时更新时间戳

destroy/1

destroy({Ets, Sizeout, Timeout, ReadTimeUpdate}) -> true

销毁缓存。

说明： 删除ETS表并释放资源。

reset/3

reset(Cache::cache(), Sizeout::integer(), Timeout::integer()) -> Cache::cache()

重置缓存参数。

参数：

  • Cache：原缓存句柄
  • Sizeout：新的内存限制
  • Timeout：新的超时时间

返回值： 新的缓存句柄

clear/1

clear({Ets, Sizeout, Timeout, ReadTimeUpdate}) -> true

清空缓存数据。

说明： 删除缓存中所有数据，但保留缓存结构。

  • Time：自定义时间戳

get/2

get(Cache::cache(), Key) -> none | {ok, Value}

获取缓存数据。

参数：

  • Key：缓存键

返回值：

  • none：键不存在或已过期
  • {ok, Value}：成功获取值

说明： 如果启用ReadTimeUpdate，会更新时间戳。

get/3

get(Cache::cache(), Key, Time::integer()) -> none | {ok, Value}

获取缓存数据（指定检查时间）。

参数：

  • Key：缓存键
  • Time：用于检查过期的时间戳

get/1

get(Cache::cache()) -> none | list()

获取缓存中所有数据。

返回值： 键值对列表 [{Key, Value, Time}, ...] 或 none

lookup/3

lookup({Ets, Sizeout, Timeout, ReadTimeUpdate}, Key, OnlyTime::boolean()) -> 
    none | Time | {Key, Value, Time}

查找缓存数据。

参数：

  • Key：缓存键
  • OnlyTime：是否只返回时间戳

返回值：

  • none：键不存在
  • Time：仅时间戳（当OnlyTime为true时）
  • {Key, Value, Time}：完整数据

delete({Ets, Sizeout, Timeout, ReadTimeUpdate}, Key) -> true

删除缓存数据。

update({Ets, Sizeout, Timeout, ReadTimeUpdate}, Key) -> none | true | false

返回值：

  • none：键不存在
  • true：更新成功
  • false：更新失败

update/3

update({Ets, Sizeout, Timeout, ReadTimeUpdate}, Key, Time) -> none | true | false

更新键的时间戳（指定时间）。

缓存维护函数

collate/1

collate({Ets, Sizeout, Timeout, ReadTimeUpdate}) -> true

整理缓存。

清理策略：

  1. 删除所有超时数据
  2. 如果内存仍然超过Sizeout，按时间段将老数据清除，每次将大小降为当前内存一半左右

collate/2

collate({Ets, Sizeout, Timeout, ReadTimeUpdate}, Time) -> true

整理缓存（指定时间阈值）。

参数：

  • Time：时间阈值，大于等于此时间的数据要保留

is_overflow/1

is_overflow(Cache::cache()) -> true | false

检查缓存是否溢出（内存超过Sizeout）。

memory_size/1

memory_size(Cache::cache()) -> true | false

获取缓存内存大小（以字为单位）。

read_time_update(Cache::cache()) -> true | false
